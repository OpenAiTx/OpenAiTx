# unplugin-check-syntax

[![npm version](https://badge.fury.io/js/@winner-fed%2Funplugin-check-syntax.svg)](https://badge.fury.io/js/@winner-fed%2Funplugin-check-syntax)

一個通用的語法檢查外掛，支援多種建構工具（Webpack、Vite、Rollup、esbuild、Rspack 等），用於檢查 JavaScript 和 HTML 檔案中的 ECMAScript 語法相容性。本外掛參考了 [rsbuild-plugin-check-syntax](https://github.com/rspack-contrib/rsbuild-plugin-check-syntax)。

## 功能特性

- 🔧 **多建構工具支援**: 支援 Webpack、Vite、Rollup、esbuild、Rspack 等
- 📝 **多檔案類型**: 支援 JavaScript 檔案與 HTML 檔案中的內嵌腳本
- 🎯 **精確定位**: 支援 sourcemap，可追溯至原始檔案位置
- ⚙️ **彈性設定**: 支援排除檔案、手動包含、錯誤類型過濾等選項
- 📁 **智慧檔案發現**: 支援 glob 模式手動包含額外的 JS/HTML 檔案
- 🌐 **Browserslist 整合**: 支援 browserslist 設定自動轉換為 ECMAScript 版本

## 安裝


```bash
npm install @winner-fed/unplugin-check-syntax --save-dev
```
## 使用方法

### Vite


```ts
// vite.config.ts
import { defineConfig } from 'vite'
import checkSyntax from '@winner-fed/unplugin-check-syntax/vite'

export default defineConfig({
  plugins: [
    checkSyntax({
      targets: ['> 1%', 'last 2 versions'],
      // 或者直接指定 ECMAScript 版本
      // ecmaVersion: 'es2015'
    }),
  ],
})
```
### Webpack


```js
// webpack.config.js
const checkSyntax = require('@winner-fed/unplugin-check-syntax/webpack')

module.exports = {

    checkSyntax({
      targets: ['> 1%', 'last 2 versions'],
    }),
  ],
}
```

## 配置選項
</translate-content>
```ts
interface CheckSyntaxOptions {
  /**
   * 目标浏览器范围，使用 browserslist 格式
   */
  targets?: string[];
  
  /**
   * 直接指定 ECMAScript 版本（优先级高于 targets）
   */
  ecmaVersion?: 'es3' | 'es5' | 'es2015' | 'es2016' | 'es2017' | 'es2018' | 'es2019' | 'es2020' | 'es2021' | 'es2022' | 'latest';
  
  /**
   * 排除源文件检查，支持正则表达式
   */
  exclude?: RegExp | RegExp[];
  
  /**
   * 排除输出文件检查，支持正则表达式  
   */
  excludeOutput?: RegExp | RegExp[];
  
  /**
   * 手动包含额外的 JS 或 HTML 文件进行语法检查
   * 支持文件路径（相对于项目根目录）或 glob 模式
   */
  include?: string[];
  
  /**
   * 排除特定的错误日志类型
   */
  excludeErrorLogs?: ('source' | 'output' | 'reason' | 'code')[];
}
```
## 使用示例

### 基礎配置


```ts
// 使用 browserslist 配置
checkSyntax({
  targets: ['> 1%', 'last 2 versions', 'not dead'],
})

// 直接指定 ECMAScript 版本
checkSyntax({
  ecmaVersion: 'es2015',
})
```
### 排除檔案


```ts
checkSyntax({
  targets: ['> 1%', 'last 2 versions'],
  // 排除 node_modules 和测试文件
  exclude: [/node_modules/, /\.test\.js$/],
  // 排除特定输出文件
  excludeOutput: [/vendor\.js$/],
})
```
### 包含額外檔案


```ts
checkSyntax({
  targets: ['> 1%', 'last 2 versions'],
  // 手动包含额外的文件进行检查
  include: [
    'src/legacy/**/*.js',        // 包含 legacy 目录下的所有 JS 文件
    'public/scripts/app.js',     // 包含特定的公共脚本文件
    'docs/**/*.html',            // 包含文档中的 HTML 文件
    '/absolute/path/to/file.js'  // 支持绝对路径
  ],
})
```
### 自訂錯誤輸出


```ts
checkSyntax({
  targets: ['> 1%', 'last 2 versions'],
  // 只显示错误原因，不显示代码片段
  excludeErrorLogs: ['code', 'output'],
})
```
## 遷移指南

如果你正在從 `@rsbuild/plugin-check-syntax` 遷移：


```diff
- import { pluginCheckSyntax } from '@rsbuild/plugin-check-syntax'
+ import checkSyntax from '@winner-fed/unplugin-check-syntax/vite' // 或其他构建工具

export default defineConfig({
  plugins: [
-   pluginCheckSyntax({
+   checkSyntax({
      targets: ['> 1%', 'last 2 versions'],
    }),
  ],
})
```
## 工作原理

1. **建構後檢查**：外掛在建構完成後檢查輸出檔案
2. **語法解析**：使用 Acorn 解析器檢查 JavaScript 語法
3. **相容性驗證**：根據配置的 ECMAScript 版本驗證語法相容性
4. **錯誤報告**：提供詳細的錯誤資訊，包括檔案位置和原始碼片段
5. **Sourcemap 支援**：如果存在 sourcemap，會追溯到原始源檔案
