# unplugin-check-syntax

[![npm version](https://badge.fury.io/js/@winner-fed%2Funplugin-check-syntax.svg)](https://badge.fury.io/js/@winner-fed%2Funplugin-check-syntax)

Ein universelles Syntax-Prüfungs-Plugin, das verschiedene Build-Tools (Webpack, Vite, Rollup, esbuild, Rspack usw.) unterstützt und zur Überprüfung der ECMAScript-Kompatibilität in JavaScript- und HTML-Dateien dient. Dieses Plugin orientiert sich an [rsbuild-plugin-check-syntax](https://github.com/rspack-contrib/rsbuild-plugin-check-syntax).

## Funktionsmerkmale

- 🔧 **Unterstützung mehrerer Build-Tools**: Unterstützt Webpack, Vite, Rollup, esbuild, Rspack usw.
- 📝 **Mehrere Dateitypen**: Unterstützt JavaScript-Dateien und Inline-Skripte in HTML-Dateien
- 🎯 **Präzise Lokalisierung**: Unterstützt Sourcemaps, kann zur Ursprungsdatei zurückverfolgen
- ⚙️ **Flexible Konfiguration**: Unterstützt Dateiausschluss, manuelles Einbinden, Fehlerarten-Filterung usw.
- 📁 **Intelligente Dateierkennung**: Unterstützt glob-Muster zum manuellen Einbinden zusätzlicher JS/HTML-Dateien
- 🌐 **Browserslist-Integration**: Unterstützt die automatische Umwandlung der browserslist-Konfiguration in eine ECMAScript-Version

## Installation


```bash
npm install @winner-fed/unplugin-check-syntax --save-dev
```
## Verwendung

### Vite


```ts
// vite.config.ts
import { defineConfig } from 'vite'
import checkSyntax from '@winner-fed/unplugin-check-syntax/vite'

export default defineConfig({
  plugins: [
    checkSyntax({
      targets: ['> 1%', 'last 2 versions'],
      // 或者直接指定 ECMAScript 版本
      // ecmaVersion: 'es2015'
    }),
  ],
})
```

## Konfigurationsoptionen


```ts
interface CheckSyntaxOptions {
  /**
   * 目标浏览器范围，使用 browserslist 格式
   */
  targets?: string[];
  
  /**
   * 直接指定 ECMAScript 版本（优先级高于 targets）
   */
  ecmaVersion?: 'es3' | 'es5' | 'es2015' | 'es2016' | 'es2017' | 'es2018' | 'es2019' | 'es2020' | 'es2021' | 'es2022' | 'latest';
  
  /**
   * 排除源文件检查，支持正则表达式
   */
  exclude?: RegExp | RegExp[];
  
  /**
   * 排除输出文件检查，支持正则表达式  
   */
  excludeOutput?: RegExp | RegExp[];
  
  /**
   * 手动包含额外的 JS 或 HTML 文件进行语法检查
   * 支持文件路径（相对于项目根目录）或 glob 模式
   */
  include?: string[];
  
  /**
   * 排除特定的错误日志类型
   */
  excludeErrorLogs?: ('source' | 'output' | 'reason' | 'code')[];
}
```
## Verwendungsbeispiel

### Grundkonfiguration


```ts
// 使用 browserslist 配置
checkSyntax({
  targets: ['> 1%', 'last 2 versions', 'not dead'],
})

// 直接指定 ECMAScript 版本
checkSyntax({
  ecmaVersion: 'es2015',
})
```
### Ausschlussdateien


```ts
checkSyntax({
  targets: ['> 1%', 'last 2 versions'],
  // 排除 node_modules 和测试文件
  exclude: [/node_modules/, /\.test\.js$/],
  // 排除特定输出文件
  excludeOutput: [/vendor\.js$/],
})
```
### Zusätzliche Dateien einbeziehen


```ts
checkSyntax({
  targets: ['> 1%', 'last 2 versions'],
  // 手动包含额外的文件进行检查
  include: [
    'src/legacy/**/*.js',        // 包含 legacy 目录下的所有 JS 文件
    'public/scripts/app.js',     // 包含特定的公共脚本文件
    'docs/**/*.html',            // 包含文档中的 HTML 文件
    '/absolute/path/to/file.js'  // 支持绝对路径
  ],
})
```
### Benutzerdefinierte Fehlerausgabe


```ts
checkSyntax({
  targets: ['> 1%', 'last 2 versions'],
  // 只显示错误原因，不显示代码片段
  excludeErrorLogs: ['code', 'output'],
})
```
## Migrationsleitfaden

Wenn du von `@rsbuild/plugin-check-syntax` migrierst:


```diff
- import { pluginCheckSyntax } from '@rsbuild/plugin-check-syntax'
+ import checkSyntax from '@winner-fed/unplugin-check-syntax/vite' // 或其他构建工具

export default defineConfig({
  plugins: [
-   pluginCheckSyntax({
+   checkSyntax({
      targets: ['> 1%', 'last 2 versions'],
    }),
  ],
})
```
## Funktionsweise

1. **Überprüfung nach dem Build**: Das Plugin überprüft die Ausgabedateien nach Abschluss des Builds
2. **Syntaxanalyse**: Überprüft die JavaScript-Syntax mit dem Acorn-Parser
3. **Kompatibilitätsprüfung**: Validiert die Syntaxkompatibilität gemäß der konfigurierten ECMAScript-Version
4. **Fehlermeldung**: Liefert detaillierte Fehlermeldungen, einschließlich Dateiposition und Quelltextausschnitt
5. **Sourcemap-Unterstützung**: Wenn eine Sourcemap vorhanden ist, wird auf die ursprüngliche Quelldatei zurückverfolgt
