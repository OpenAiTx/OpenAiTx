# Beken7252 OpenCam
A9カメラ向けRT-Thread代替プロジェクト

# 更新情報
## 25/09/25
修士課程を修了し、少し自由な時間ができました。これらのカメラはまだ多く残っているので、このプロジェクトの最初の正式リリースを計画する時期です。数週間取り組んでおり、現在は起動し、Wi-Fiに接続し、SDカードの読み書きを行い、MQTTをサポートし、最も重要なことに少なくとも3種類の異なるカメラセンサー：GC0311、HI704、GC0328で動作する初期バージョンがあります。これには、写真の保存とHTTP（multipart/x-mixed-replace）を用いたストリーミングも含まれます。

安定性のための追加テストやSDKおよびプロジェクト全体の状態の改善がまだ必要です。

詳細な1.0.0バージョンの計画は[このチケット](https://github.com/daniel-dona/beken7252-opencam/issues/21)を参照してください。

## 20/11/2023
先月、追加のA9カメラの到着を待っていました。手元には1台しかなく、テスト用ワイヤの頻繁なはんだ付け・はんだ除去で基板を壊すのを心配していました。
しかし、期待通りにはいきませんでした。Aliexpressで約3€だったので、12台ほどの異なるカメラを手に入れました。

~6台がBekenチップ搭載

~3台がカスタムXRadio/Allwinnerのように見える不明なチップ

~1台が記憶にない他のベンダーのもの

~1台が何の刻印もないチップ

数字はあまり合いませんが、重要なのはBekenチップ搭載のものが増えたことです…本当でしょうか？

チップにはBK7252と刻印されていますが、ピン配置が異なります。文書化されていて少なくともデータシートがあるチップは68ピンですが、これは48ピンでリファレンスはBK7252UQN48です。

詳細はこちら：https://github.com/daniel-dona/beken7252-opencam/wiki/BK7252-SoC-variations

しばらくはこのチップで作業しません。動作するコードをアップロードしようとしましたが、期待通りに動作しません。確実に何らかの機能が削除されているか、ピンマルチプレクサが異なるかもしれません。あるいはメモリ領域のレジスタが違うのかもしれません。あるいは私が愚かでコードのアップロードを間違えたのかも…

いずれにせよ、まずは既知のチップで動作するものを作ることに集中し、その後この新しいチップを検討するのが良いと思います。
