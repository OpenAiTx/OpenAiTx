마지막 업데이트: *2026년 1월 25일 (버전 1.2)*  
![부채꼴 배열로 펼쳐진 WCAG 2.2 카드 덱의 4장 카드. 카드에는 기준의 간단한 설명, 장애 및 책임을 나타내는 아이콘, 전체 설명에 대한 링크와 QR 코드가 포함되어 있습니다. 카드 덱 버전 0.4와 크리에이티브 커먼즈 라이선스 CC BY-SA 4.0이 하단에 보입니다.](https://raw.githubusercontent.com/johanneslehner/wcag2.2-card-deck/main/public/images/Cover.png)  
[WCAG 2.2 카드 덱의 피그마 커뮤니티 파일](https://www.figma.com/community/file/1409436654182046971/wcag-2-2-card-deck)로 이동  
  
  
## 이 카드 덱은 무엇인가요?  
WCAG 2.2 카드 덱은 여러 훌륭한 접근성 자료를 하나의 도구 키트로 결합하기 위한 주말 프로젝트로 시작되었습니다:  
- 모든 WCAG 2.2 성공 기준 요약  
- 이해하기 쉬운 설명으로  
- 실용적인 테스트 주제별로 그룹화 (Andrew Hick의 '[WCAG 2.2 주제별 지도](https://andrewhick.com/accessibility/wcag-map/)', '[WCAG 2.2 주제별 지도 - AAA 버전](https://andrewhick.com/aaa/)' 및 '[WCAG 결정 트리](https://github.com/andrewhick/accessibility/wiki/WCAG-decision-tree)' 기반)  
- 각 기준을 충족했을 때 혜택을 받는 대상 표시 ([Atomic A11Y](https://www.atomica11y.com/)의 Charlie Triplett, [Appt.org](https://appt.org/en/), [IBM 접근성 요구사항](https://www.ibm.com/able/requirements/requirements/) 기반)  
- 주로 책임이 있는 역할 강조 (GOV.UK [WCAG 프라이머](https://alphagov.github.io/wcag-primer/), 멜버른 대학교의 '[프로젝트 역할별 접근성 책임](https://www.unimelb.edu.au/accessibility/project-management-old/responsibility-breakdown/print-version)' 체크리스트 기반)  
- 함께 작동하기 좋은 성공 기준 제안  
  
  
## 어떻게 사용하나요?  
WCAG 2.2 카드 덱을 워크숍, 접근성 감사, 교육 등 다양한 용도로 사용하세요.  
  
  
## 다음 계획은 무엇인가요?  
LinkedIn 초기 공지 후 받은 훌륭한 피드백 덕분에 앞으로 추가할 기능 목록이 이미 있습니다:  
- 더 많은 언어 (이미 계획된 언어: 일본어, 아랍어, 스웨덴어, 노르웨이어, 카탈란어 - 추가 예정)  
- 방법 설명 추가 (카드 덱을 최대한 활용하는 방법 - 예: 교육/학습, 접근성 감사용)  
- 각 성공 기준 테스트에 사용할 수 있는 도구 추가 (관련 외부 출처 포함)  
- 카드 덱과 함께 사용할 수 있는 도구에 대한 참조 포함  
- 접근 가능한 HTML 버전의 WCAG 2.2 카드 덱 (공개 GitHub 저장소에; CC BY-SA 4.0 라이선스 하에)  
- WCAG 카드 덱 포스터  
- '주제별 WCAG 2.2 지도' 포스터 (지하철 노선도 형식으로 카드 시각화)  
- 각 성공 기준에 대한 좋은/나쁜 예시 (독립적 향상 또는 카드 활용 교육 지원용)  
- 무시할 경우 발생하는 장애 정보  
- ... 기타
