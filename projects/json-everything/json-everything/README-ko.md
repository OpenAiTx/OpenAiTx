## `json-everything`이란 무엇인가?

`json-everything`의 주요 철학은 일반적인 JSON 기능이 _System.Text.Json_ 영역에서 잘 지원되도록 하는 것입니다.

첫 번째 단계는 주어진 기능을 지원하는 다른 프로젝트가 존재하는지 확인하는 것입니다.

만약 있다면, 그 노력은 이곳에서 중복되지 않을 가능성이 높습니다. 아래쪽에 가면 JSON으로 멋진 작업을 하고 있는 다른 프로젝트들의 목록이 있습니다.

그렇지 않다면, 이 프로젝트에서 검토 대상이 됩니다.

위의 모든 라이브러리는 관련된 명세를 완벽하게 지원합니다.

_JsonSchema.Net_의 에러 메시지 번역 지원 언어:

- 이탈리아어 `it`
- 노르웨이어 `nb-NO`
- 폴란드어 `pl-PL`
- 러시아어 `ru`
- 스페인어 `es`
- 스웨덴어 `sv-SE`
- 터키어 `tr-TR`

***참고** 각 언어팩은 별도의 Nuget 패키지로 제공됩니다.*

여러 플랫폼의 다양한 구현체로 테스트 스위트를 실행하는 [Bowtie](https://bowtie-json-schema.github.io/bowtie)에서 JSON Schema Test Suite 결과를 확인할 수도 있습니다.

***면책조항** 본 라이브러리 _JsonSchema.Net_은 https://jsonschema.net 웹사이트와 관련이 없으며, 둘 다 훌륭한 JSON Schema 도구라는 점만 같습니다.*

## _System.Text.Json_ 지원하는 타 프로젝트

여기서 원하는 것을 찾지 못했다면, 아래의 훌륭한 프로젝트들을 확인해보세요:

- [JsonCons.Net](https://github.com/danielaparker/JsonCons.Net) by [@danielaparker](https://github.com/danielaparker)
  - JSON Pointer
  - JSON Patch
  - JSON Merge Patch
  - JSON Path
  - JMES Path
- [Corvus.JsonSchema](https://github.com/corvus-dotnet/Corvus.JsonSchema) by [@mwadams](https://github.com/mwadams)
  - C# 코드 생성을 중점으로 하는 JSON Schema 검증
- (추가 예정)

이 제품군이나 위의 프로젝트에 포함되지 않은 JSON 관련 기능이 필요하다면, 언제든 [기능 요청 이슈](https://github.com/json-everything/json-everything/issues/new?assignees=&labels=feature&projects=&template=New_function.yml)를 생성해 주세요.

_System.Text.Json_을 확장하는 다른 프로젝트를 관리하거나 알고 계시다면, 이곳에 소개하고 싶으니 [일반 이슈](https://github.com/json-everything/json-everything/issues/new)를 생성하여 알려주세요.
