## `json-everything`とは？

`json-everything`の主な思想は、一般的なJSON機能に対して_System.Text.Json_空間で十分なサポートを提供することです。

その最初のステップは、特定の機能をサポートする他のプロジェクトが存在するかどうかを確認することです。

もし存在すれば、その取り組みをここで重複することはおそらくありません。もう少し下に進むと、JSONに関して素晴らしいことをしている他のプロジェクトのリストがあります。

存在しなければ、本プロジェクトでの検討対象となります。

上記のすべてのライブラリは、関連する仕様への完全なサポートを提供しています。

_JsonSchema.Net_ のエラーメッセージ翻訳は以下の言語で利用可能です:

- イタリア語 `it`
- ノルウェー語 `nb-NO`
- ポーランド語 `pl-PL`
- ポルトガル語 `pt`
- ロシア語 `ru`
- スペイン語 `es`
- スウェーデン語 `sv-SE`
- トルコ語 `tr-TR`

***注意** 各言語パックは、それぞれ独自のNugetパッケージとして提供されています。*

また、[Bowtie](https://bowtie-json-schema.github.io/bowtie)でJSON Schema Test Suiteの結果を見ることができます。Bowtieは、複数のプラットフォームにわたる様々な実装に対してテストスイートを実行します。

***免責事項** 私のライブラリ _JsonSchema.Net_ は、https://jsonschema.net というウェブサイトとは関係ありませんが、どちらも優れたJSON Schemaツールです。*

## 他プロジェクトによる _System.Text.Json_ サポート

ここでお探しのものが見つからない場合、次の優れたプロジェクトもご利用ください:

- [JsonCons.Net](https://github.com/danielaparker/JsonCons.Net) （[@danielaparker](https://github.com/danielaparker) による）

  - JSON Pointer
  - JSON Patch
  - JSON Merge Patch
  - JSON Path
  - JMES Path
- [Corvus.JsonSchema](https://github.com/corvus-dotnet/Corvus.JsonSchema) （[@mwadams](https://github.com/mwadams)による）
  - C#のコード生成に重点を置いたJSON Schemaバリデーション
- （今後さらに追加予定）

もしJSONを使ってこのスイートや上記プロジェクトでカバーされていないことをしている場合は、[機能追加のIssueを作成](https://github.com/json-everything/json-everything/issues/new?assignees=&labels=feature&projects=&template=New_function.yml)してください。

_System.Text.Json_を拡張して便利なことをしている他のプロジェクトを知っていたり、メンテナンスしている場合は、ここに掲載したいので、[一般的なIssueを作成](https://github.com/json-everything/json-everything/issues/new)してお知らせください。
