## Что такое `json-everything`?

Основная философия `json-everything` заключается в обеспечении хорошей поддержки распространённого функционала JSON в пространстве _System.Text.Json_.

Первый шаг — это проверка наличия других проектов, поддерживающих нужную функцию.

Если такие проекты существуют, то усилия, скорее всего, не будут дублироваться здесь. Чуть ниже вы найдёте список других проектов, реализующих интересные возможности с JSON.

В противном случае функция рассматривается для включения в этот проект.

Переводы сообщений об ошибках для _JsonSchema.Net_ доступны на следующих языках:

- Итальянский `it`
- Норвежский `nb-NO`
- Польский `pl-PL`
- Португальский `pt`
- Русский `ru`
- Испанский `es`
- Шведский `sv-SE`
- Турецкий `tr-TR`

***ПРИМЕЧАНИЕ** Каждый языковой пакет предоставляется отдельным Nuget-пакетом.*

Вы также можете просмотреть результаты тестового набора JSON Schema на [Bowtie](https://bowtie-json-schema.github.io/bowtie), который запускает тесты для различных реализаций на разных платформах.

***ОТКАЗ ОТ ОТВЕТСТВЕННОСТИ** Моя библиотека, _JsonSchema.Net_, не связана и не ассоциирована с сайтом https://jsonschema.net, кроме того, что оба являются отличными инструментами для работы с JSON Schema.*

  - Валидация JSON Schema с акцентом на генерацию C# кода
- (дополнения будут позже)

Если вы используете JSON для чего-то, что не охвачено библиотеками из этого набора или из указанных выше проектов, не стесняйтесь [создать issue с предложением функции](https://github.com/json-everything/json-everything/issues/new?assignees=&labels=feature&projects=&template=New_function.yml).

Если вы поддерживаете или знаете другой проект, который расширяет _System.Text.Json_ для интересных задач, я хотел бы добавить его сюда, пожалуйста, [создайте общее issue](https://github.com/json-everything/json-everything/issues/new) и сообщите мне о нем.
