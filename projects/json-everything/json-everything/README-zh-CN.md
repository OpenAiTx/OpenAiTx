## 什么是 `json-everything`？

`json-everything` 背后的主要理念是确保常见的 JSON 功能在 _System.Text.Json_ 领域中得到良好支持。

实现这一目标的第一步是检查是否存在支持某项功能的其他项目。

如果有，这项工作可能不会在此重复。稍后你会看到一些正在用 JSON 做很酷事情的其他项目列表。

否则，这项功能将开放考虑纳入本项目。

_JsonSchema.Net_ 的错误消息翻译支持以下语言：

- 意大利语 `it`
- 挪威语 `nb-NO`
- 波兰语 `pl-PL`
- 葡萄牙语 `pt`
- 俄语 `ru`
- 西班牙语 `es`
- 瑞典语 `sv-SE`
- 土耳其语 `tr-TR`

***注意** 每个语言包均通过其独立的 Nuget 包提供。*

您还可以在 [Bowtie](https://bowtie-json-schema.github.io/bowtie) 上查看 JSON Schema 测试套件的结果，该平台在不同平台的多个实现上运行测试套件。

## 其他项目对 _System.Text.Json_ 的支持

如果你在这里没有找到所需内容，请尝试以下优秀项目之一：

- [JsonCons.Net](https://github.com/danielaparker/JsonCons.Net) 作者 [@danielaparker](https://github.com/danielaparker)

  - JSON 指针
  - JSON 补丁
  - JSON 合并补丁
  - JSON 路径
  - JMES 路径
- [Corvus.JsonSchema](https://github.com/corvus-dotnet/Corvus.JsonSchema) 作者为 [@mwadams](https://github.com/mwadams)
  - 以 C# 代码生成为重点的 JSON Schema 校验
- （更多内容即将推出）

如果你使用 JSON 做一些该套件或上述项目未涵盖的事情，欢迎[创建功能问题](https://github.com/json-everything/json-everything/issues/new?assignees=&labels=feature&projects=&template=New_function.yml)。

如果你维护或知道另一个扩展 _System.Text.Json_ 实现酷功能的项目，我想在这里列出它，所以请[创建一般问题](https://github.com/json-everything/json-everything/issues/new)让我知道。
