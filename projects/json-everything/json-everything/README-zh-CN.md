## 什么是 `json-everything`？

`json-everything` 背后的主要理念是在 _System.Text.Json_ 领域内确保常见的 JSON 功能获得良好的支持。

实现这一目标的第一步是检查是否已经有其他项目支持某项功能。

如果已经有，那么这里可能不会重复造轮子。往下看你会发现一些正在用 JSON 做很酷事情的其它项目列表。

否则，这个功能就有可能被考虑加入本项目。

以上所有库都完全支持各自关联的规范。

_JsonSchema.Net_ 的错误信息翻译现已支持：

- 意大利语 `it`
- 挪威语 `nb-NO`
- 波兰语 `pl-PL`
- 俄语 `ru`
- 西班牙语 `es`
- 瑞典语 `sv-SE`
- 土耳其语 `tr-TR`

***注意** 每种语言包都由其各自的 Nuget 包单独提供。*

你还可以在 [Bowtie](https://bowtie-json-schema.github.io/bowtie) 上查看 JSON Schema 测试套件的结果，该平台会在多个平台的不同实现上运行测试套件。

***免责声明** 我的库 _JsonSchema.Net_ 与 https://jsonschema.net 网站没有任何关系或关联，除了它们都是出色的 JSON Schema 工具。*

## 其他项目对 _System.Text.Json_ 的支持

如果你在这里没有找到所需内容，请试试以下这些优秀项目：

- [JsonCons.Net](https://github.com/danielaparker/JsonCons.Net) by [@danielaparker](https://github.com/danielaparker)
  - JSON Pointer
  - JSON Patch
  - JSON Merge Patch
  - JSON Path
  - JMES Path
- [Corvus.JsonSchema](https://github.com/corvus-dotnet/Corvus.JsonSchema) by [@mwadams](https://github.com/mwadams)
  - 以 C# 代码生成为重点的 JSON Schema 校验
- （后续还会有更多）

如果你用 JSON 做了某些本套件或上述项目未涵盖的事情，欢迎[创建功能请求](https://github.com/json-everything/json-everything/issues/new?assignees=&labels=feature&projects=&template=New_function.yml)。

如果你维护或了解其他扩展 _System.Text.Json_ 的项目，并且它很酷，我很乐意将其列在这里，请[创建一般问题](https://github.com/json-everything/json-everything/issues/new)让我知道。
