# unused-task-warning
의존성 주입과 async-await 패턴을 사용할 때, Task를 반환하는 메서드를 가진 인터페이스가 생길 수 있습니다. 이 인터페이스 메서드가 동기 메서드 내에서 사용되면, 내장 경고 CS4014를 발생시키지 않는 fire and forget 방식으로 잘못 실행될 가능성이 있습니다. 이 경우 이 분석기는 경고를 생성합니다.

둘 다 Visual Studio 확장으로 사용 가능하며, NuGet에서 제공되는 프로젝트 분석기로 사용하는 것이 더 바람직합니다.

예제:

	using System.Threading.Tasks;

	namespace AsyncAwaitProblem
	{
		public interface ICallee
		{
			bool ProblemSolved { get; }
			Task SolveProblemAsync();
		}

		public class Callee : ICallee
		{
			public bool ProblemSolved { get; set; }

			public async Task SolveProblemAsync()
			{
				await Task.Delay(10);
				ProblemSolved = true;
			}
		}
		
		public class Caller
		{
			public bool DoCall()
			{
				ICallee xxx = new Callee();
	
				// 이 분석기는 다음 줄에서 경고를 표시합니다.
				xxx.SolveProblemAsync(); // 이것은 아마도 원치 않는 fire and forget일 것입니다.
	
				return xxx.ProblemSolved; // false를 반환하며, true를 기대했습니다.
			}
		}
	}


이 분석기는 현재 알려진 대기 가능한 유형(예: `Task`, `ValueTask` 및 `ConfigureAwait` 메서드 사용 시 반환되는 `ConfiguredTaskAwaitable` 유형)만 검사합니다. 다른 ['Awaitable'](https://blogs.msdn.microsoft.com/pfxteam/2011/01/13/await-anything/) 유형이 반환되면 이 분석기는 경고를 표시하지 않습니다. 이는 향후 버전에서 수정될 수 있습니다.

NuGet에서 사용 가능: https://www.nuget.org/packages/Lindhart.Analyser.MissingAwaitWarning/

Visual Studio 마켓플레이스에서 사용 가능: https://marketplace.visualstudio.com/items?itemName=Lindhart.missingAwaitWarning#overview

분석기 사용 방법에 대한 정보는 여기에서 확인하세요: https://docs.microsoft.com/en-us/visualstudio/code-quality/roslyn-analyzers-overview?view=vs-2019
