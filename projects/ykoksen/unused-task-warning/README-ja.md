# unused-task-warning
依存性注入とasync-awaitパターンを使用すると、Taskを返すメソッドを持つインターフェイスが作成されることがあります。このインターフェイスのメソッドが同期メソッド内で使用される場合、それが誤ってファイアアンドフォゲット方式で実行される可能性があります（これは組み込みの警告CS4014をトリガーしません）。このような状況で、このアナライザーは警告を生成します。

Visual Studioの拡張機能として、またはNuGetから入手可能なプロジェクトアナライザーとして使用可能です。

例:

	using System.Threading.Tasks;

	namespace AsyncAwaitProblem
	{
		public interface ICallee
		{
			bool ProblemSolved { get; }
			Task SolveProblemAsync();
		}

		public class Callee : ICallee
		{
			public bool ProblemSolved { get; set; }

			public async Task SolveProblemAsync()
			{
				await Task.Delay(10);
				ProblemSolved = true;
			}
		}
		
		public class Caller
		{
			public bool DoCall()
			{
				ICallee xxx = new Callee();
	
				// このアナライザーは次の行で警告を表示します
				xxx.SolveProblemAsync(); // これはおそらく望ましくないファイアアンドフォゲットです。
	
				return xxx.ProblemSolved; // falseを返します - trueを返すことを期待していました
			}
		}
	}


このアナライザーは現在、既知の待機可能な型（例：`Task`、`ValueTask`、および `ConfigureAwait` メソッド使用時に返される型である `ConfiguredTaskAwaitable`）のみをチェックします。別の ['Awaitable'](https://blogs.msdn.microsoft.com/pfxteam/2011/01/13/await-anything/) 型が返された場合、このアナライザーは警告を表示しません。これは将来のバージョンで修正される可能性があります。

NuGetから入手可能：https://www.nuget.org/packages/Lindhart.Analyser.MissingAwaitWarning/

Visual Studioマーケットプレイスから入手可能：https://marketplace.visualstudio.com/items?itemName=Lindhart.missingAwaitWarning#overview

アナライザーの使用方法については、こちらをご覧ください：https://docs.microsoft.com/en-us/visualstudio/code-quality/roslyn-analyzers-overview?view=vs-2019
