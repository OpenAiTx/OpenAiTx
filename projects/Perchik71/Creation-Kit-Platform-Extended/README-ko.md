# 설명
**CKPE**는 공유 소스 기반의 강력한 플랫폼으로, **Bethesda**의 Creation Kit 에디터에 수많은 수정과 개선을 제공합니다. **Skyrim Special Edition**, **Fallout 4**, **Starfield** 등 게임의 에디터 지원을 포함하며, [SSE CKFixes](https://github.com/Nukem9/skyrimse-test), [SSE CKFixes Update](https://github.com/Perchik71/SkyrimSETest), [FO4 CKFixes](https://github.com/Perchik71/Fallout4Test) 프로젝트의 후속작이자, **perchik71** 저자의 모든 시작점이 된 [Unicode patch for SSE CK 1.5.73](https://github.com/Perchik71/usse_test)도 포함합니다.

# 설치
>[!중요]
>정규 또는 noavx2 릴리즈?  
>Steam에서 CPU가 avx2를 지원하는지 확인하세요. Steam을 열고 왼쪽 상단의 도움말 메뉴에 마우스를 올린 후 시스템 정보를 선택하고 AVX2를 검색하세요.  
>지원된다면 정규 릴리즈를 선택하세요  
>지원되지 않는다면 noavx2 릴리즈를 선택하세요  
### 신규 설치
[지원 게임](https://github.com/Perchik71/Creation-Kit-Platform-Extended/wiki#brief) 중 아무거나 다운로드 후 Creation Kit을 설치하세요.  
해당 게임용 **CKPE**의 [최신 릴리즈](https://github.com/Perchik71/Creation-Kit-Platform-Extended/releases)를 다운로드하세요.  
모든 파일을 게임 디렉토리에 압축 해제하여 **ckpe_loader.exe**가 **CreationKit.exe**와 동일한 폴더에 있도록 하세요.
### 업데이트
해당 게임용 **CKPE**의 [최신 릴리즈](https://github.com/Perchik71/Creation-Kit-Platform-Extended/releases)를 다운로드하세요.  
모든 파일을 게임 디렉토리에 압축 해제하여 **ckpe_loader.exe**가 **CreationKit.exe**와 동일한 폴더에 있도록 하세요.  
모든 파일 교체에 동의하세요.
### 컴파일
일반 사용자는 아직 출시되지 않은 최신 기능에 접근할 때만 이 방법이 필요합니다.  
**`Visual Studio 2022`** 이상에서 프로젝트를 열고 **`Release`** 또는 **`Release-NoAVX2`** 모드로 빌드하세요.  
빌드 결과물을 게임 디렉토리로 옮겨 **ckpe_loader.exe**가 **CreationKit.exe**와 동일한 폴더에 있도록 하세요.  
기존 파일 교체 요청 시 모두 교체에 동의하세요.
### 빌드 자동화 (베타)
해당 게임용 **CKPE**의 [최신 커밋 빌드](https://github.com/Perchik71/Creation-Kit-Platform-Extended/actions)를 다운로드하세요.  
모든 파일을 게임 디렉토리에 압축 해제하여 **ckpe_loader.exe**가 **CreationKit.exe**와 동일한 폴더에 있도록 하세요.  
발생 가능한 오류.
