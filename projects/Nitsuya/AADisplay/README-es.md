Hacer que Android Auto muestre cualquier aplicación, basado en un esquema de VirtualDisplay anidado, ¡Star! ¡Star! ¡Star!

Solo compatible con A12+, requiere LSPosed, algunas ROM pueden tener posibilidad de fallos, por favor tenga conocimientos básicos para recuperación.

Teóricamente puede soportar hasta A10+, pero debido a la falta de dispositivos y tiempo para mantenimiento, no se soporta; si tiene capacidad, haga un PR.

Por favor no envíe Issues relacionados con problemas ajenos al proyecto, ¡serán cerrados! Por ejemplo: instalación de Magisk/LSPosed, problemas de conexión/uso de AA, etc.

## Uso
- Active este módulo en LSPosed, marque System Framework y Android Auto, instale su lanzador favorito y complete el nombre del paquete en la configuración del módulo.
- Se recomienda configurar el DPI, mejorará la experiencia con aplicaciones que dibujan UI usando el contexto de la aplicación.
- Agregue propiedades relacionadas con AA para modificar la configuración mediante hooks, solo se soportan parámetros de tipo básico.
- Se omite automáticamente la verificación de la fuente de instalación de la aplicación AA.
- El permiso Root solo se usa para ejecutar comandos Shell configurados por el usuario, si no es necesario, no lo otorgue.
