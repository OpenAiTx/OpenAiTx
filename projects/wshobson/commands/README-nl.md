**52 commando's** georganiseerd als:

### ü§ù Vereist Claude Code Subagents

Deze commando's werken met de [Claude Code Subagents](https://github.com/wshobson/agents) voor orkestratiemogelijkheden.

## Installatie

```bash
cd ~/.claude
git clone https://github.com/wshobson/commands.git
```

## Command Format

Slash-commando‚Äôs zijn eenvoudige markdown-bestanden waarbij:
- De bestandsnaam de commando-naam wordt (bijv. `api-scaffold.md` ‚Üí `/api-scaffold`)
- De inhoud van het bestand de prompt/instructies zijn die worden uitgevoerd bij aanroepen
- Gebruik de `$ARGUMENTS` placeholder voor gebruikersinvoer

## Best Practices

### Commandoselectie
- **Laat Claude Code automatisch voorstellen** - Analyseert de context en kiest optimale commando‚Äôs
- **Gebruik workflows voor complexe taken** - Subagenten co√∂rdineren implementaties over domeinen
- **Gebruik tools voor gerichte taken** - Pas specifieke commando‚Äôs toe voor doelgerichte verbeteringen

### Effectief Gebruik
- **Geef uitgebreide context** - Vermeld tech stack, beperkingen en vereisten
- **Koppel commando‚Äôs strategisch** - Workflows ‚Üí Tools ‚Üí Verbeteringen
- **Bouw voort op eerdere output** - Commando‚Äôs zijn ontworpen om samen te werken

## Bijdragen

1. Maak een `.md`-bestand in `workflows/` of `tools/`
2. Gebruik kleine letters-met-koppelteken-namen
3. Voeg `$ARGUMENTS` toe voor gebruikersinvoer

## Problemen Oplossen

**Commando niet gevonden**: Controleer of bestanden in `~/.claude/commands/` staan

**Workflows traag**: Normaal - ze co√∂rdineren meerdere subagenten

**Algemene output**: Voeg meer specifieke context toe over je tech stack

**Integratieproblemen**: Controleer bestandslocaties en commando-volgorde

## Prestatietips

**Commando-selectie:**
- **Workflows**: Co√∂rdinatie van meerdere subagenten voor complexe features

- Begin met tools voor bekende problemen
