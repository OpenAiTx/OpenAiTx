# Claude Kod Alt Ajanları Koleksiyonu

[Claude Code](https://docs.anthropic.com/en/docs/claude-code) için geliştirilmiş, alanında uzmanlaşmış AI alt ajanlarının kapsamlı koleksiyonu; geliştirme iş akışlarını alan özel uzmanlıkla güçlendirmek için tasarlanmıştır.

## Genel Bakış

Bu depo, Claude Code'un yeteneklerini genişleten 57 adet uzmanlaşmış alt ajan içerir. Her alt ajan belirli bir alanda uzmandır; bağlama göre otomatik olarak devreye girer veya ihtiyaç halinde doğrudan çağrılabilir. Tüm ajanlar, görev karmaşıklığına göre belirli Claude modelleriyle yapılandırılmıştır; böylece en iyi performans ve maliyet etkinliği sağlanır.

## Kullanım

### Otomatik Çağrı
Claude Code, görev bağlamına ve alt ajan açıklamasına göre uygun alt ajana otomatik olarak yönlendirme yapacaktır.

### Açık Çağrı
Talebinizde alt ajanı adıyla belirtin:
```
"Use the code-reviewer to check my recent changes"
"Have the security-auditor scan for vulnerabilities"
"Get the performance-engineer to optimize this bottleneck"
```

```bash
# Feature development workflow

# Product launch workflow

# Automatically uses: business-analyst → content-marketer → sales-automator → customer-support
```

```bash
# Complex feature development (8+ subagents)

# Production incident response (5+ subagents)

```

## Alt Ajan Formatı

Her alt ajan şu yapıyı takip eder:
```markdown
---
name: subagent-name
description: When this subagent should be invoked
model: haiku  # Optional - specify which model to use (haiku/sonnet/opus)
tools: tool1, tool2  # Optional - defaults to all tools
---

System prompt defining the subagent's role and capabilities
```

### Model Yapılandırması

Claude Code v1.0.64 itibarıyla, alt ajanlar hangi Claude modelini kullanacaklarını belirtebilirler. Bu, karmaşıklığa göre maliyet etkin görev dağılımı sağlar:

- **Düşük Karmaşıklık (Haiku)**: Basit veri analizi, dokümantasyon oluşturma ve standart yanıtlar gibi basit görevler
- **Orta Karmaşıklık (Sonnet)**: Geliştirme görevleri, kod incelemesi, test ve standart mühendislik çalışmaları  
- **Yüksek Karmaşıklık (Opus)**: Güvenlik denetimi, mimari inceleme, olay müdahalesi ve AI/ML mühendisliği gibi kritik görevler

Mevcut modeller (Claude Code v1.0.64 itibarıyla sadeleştirilmiş adlandırma kullanılarak):
- `haiku` - Basit görevler için hızlı ve maliyet etkin
- `sonnet` - Çoğu geliştirme işi için dengeli performans
- `opus` - Karmaşık analizler ve kritik görevler için en yetenekli

Model belirtilmezse, alt ajan sistemin varsayılan modelini kullanır.

## Ajan Orkestrasyon Desenleri

Claude Code, bu yaygın desenleri kullanarak ajanları otomatik olarak koordine eder:

### Sıralı İş Akışları
```
User Request → Agent A → Agent B → Agent C → Result

Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor
```

### Parallel Execution
```
User Request → Agent A + Agent B (simultaneously) → Merge Results

Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations
```

### Koşullu Dallanma
```
User Request → Analysis → Route to appropriate specialist

Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter
```

### İnceleme & Doğrulama
```
Primary Agent → Review Agent → Final Result

Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation
```

## Sorun Giderme

### Yaygın Sorunlar

**Temsilci otomatik olarak devreye alınmıyor:**
- Talebinizin alanı net şekilde belirttiğinden emin olun (örn. "performans sorunu" → performans-mühendisi)
- Görev türü konusunda spesifik olun (örn. "kodu gözden geçir" → kod-inceleyici)

**Beklenmeyen temsilci seçimi:**
- Teknoloji yığınınız ve gereksinimleriniz hakkında daha fazla bağlam sağlayın
- Belirli bir temsilciye ihtiyacınız varsa açık bir çağrı kullanın

**Birden fazla temsilci çelişkili önerilerde bulunuyor:**
- Bu normaldir - farklı uzmanlar farklı önceliklere sahip olabilir
- Açıklama isteyin: "security-auditor ve performance-engineer'ın önerilerini uzlaştır"

**Temsilci bağlamdan yoksun görünüyor:**
- Talebinizde arka plan bilgisi sağlayın
- Önceki konuşmalara veya yerleşik kalıplara referans verin

### Yardım Alma

Temsilciler beklenildiği gibi çalışmıyorsa:
1. Her bir temsilcinin kendi dosyasındaki açıklamalarını kontrol edin
2. Taleplerinizde daha spesifik bir dil kullanmayı deneyin
3. Belirli temsilcileri test etmek için açık çağrı kullanın
4. Projeniz ve hedefleriniz hakkında daha fazla bağlam sağlayın
