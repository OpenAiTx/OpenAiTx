# مجموعة الوكلاء الفرعيين لـ Claude Code

مجموعة شاملة من الوكلاء الفرعيين الذكاء الاصطناعي المتخصصين لـ [Claude Code](https://docs.anthropic.com/en/docs/claude-code)، مصممة لتعزيز سير عمل التطوير بخبرات متخصصة في مجالات محددة.

## نظرة عامة

يحتوي هذا المستودع على 58 وكيلاً فرعياً متخصصاً توسع من قدرات Claude Code. كل وكيل فرعي خبير في مجال معين، ويتم استدعاؤه تلقائياً بناءً على السياق أو عند الطلب عند الحاجة. جميع الوكلاء مهيئون بنماذج Claude محددة حسب تعقيد المهمة لتحقيق أفضل أداء وتكلفة فعّالة.

### إعداد النموذج

اعتبارًا من Claude Code v1.0.64، يمكن للوكلاء الفرعيين تحديد نموذج Claude الذي يجب عليهم استخدامه. يسمح ذلك بتفويض المهام بشكل فعال من حيث التكلفة استنادًا إلى التعقيد:

- **تعقيد منخفض (Haiku)**: مهام بسيطة مثل تحليل البيانات الأساسي، إنشاء الوثائق، والاستجابات القياسية
- **تعقيد متوسط (Sonnet)**: مهام التطوير، مراجعة الكود، الاختبار، والعمل الهندسي القياسي  
- **تعقيد عالٍ (Opus)**: مهام حرجة مثل تدقيق الأمان، مراجعة البنية، الاستجابة للحوادث، وهندسة الذكاء الاصطناعي/تعلم الآلة

النماذج المتوفرة (باستخدام التسميات المبسطة اعتبارًا من Claude Code v1.0.64):
- `haiku` - سريع وفعّال من حيث التكلفة للمهام البسيطة
- `sonnet` - أداء متوازن لمعظم أعمال التطوير
- `opus` - الأكثر قدرة على التحليل المعقد والمهام الحرجة

إذا لم يتم تحديد نموذج، سيستخدم الوكيل الفرعي النموذج الافتراضي للنظام.

## أنماط تنظيم الوكلاء

يقوم Claude Code تلقائيًا بتنسيق الوكلاء باستخدام هذه الأنماط الشائعة:

### سير العمل التسلسلي
```
User Request → Agent A → Agent B → Agent C → Result

Example: "Build a new API feature"
backend-architect → frontend-developer → test-automator → security-auditor
```

### Parallel Execution
```
User Request → Agent A + Agent B (simultaneously) → Merge Results

Example: "Optimize application performance" 
performance-engineer + database-optimizer → Combined recommendations
```

### التفرع الشرطي
```
User Request → Analysis → Route to appropriate specialist

Example: "Fix this bug"
debugger (analyzes) → Routes to: backend-architect OR frontend-developer OR devops-troubleshooter
```

### المراجعة والتحقق
```
Primary Agent → Review Agent → Final Result

Example: "Implement payment processing"
payment-integration → security-auditor → Validated implementation
```

## متى تستخدم أي وكيل

### 🏗️ التخطيط والهندسة المعمارية
- **backend-architect**: تصميم واجهات البرمجة (API)، مخططات قواعد البيانات، هندسة النظام
- **frontend-developer**: تخطيط واجهة المستخدم وتجربة المستخدم (UI/UX)، هندسة المكونات
- **ui-ux-designer**: تصميم الواجهات، الرسومات التخطيطية، أنظمة التصميم، بحث المستخدمين
- **cloud-architect**: تصميم البنية التحتية، تخطيط قابلية التوسع

### 🔧 التنفيذ والتطوير  
- **python-pro**: مهام التطوير الخاصة بلغة بايثون
- **golang-pro**: مهام التطوير الخاصة بلغة Go
- **rust-pro**: تطوير Rust، أمان الذاكرة، برمجة الأنظمة
- **c-pro**: برمجة بلغة C، الأنظمة المدمجة، الشيفرات ذات الأداء العالي
- **javascript-pro**: جافاسكريبت الحديثة، الأنماط غير المتزامنة، كود Node.js/المتصفح
- **typescript-pro**: تايب سكريبت المتقدمة، الأنواع العامة، استنتاج الأنواع، أنماط المؤسسات
- **java-pro**: تطوير جافا الحديثة، التدفقات، التزامن، Spring Boot
- **elixir-pro**: تطوير Elixir، أنماط OTP، أطر Phoenix، البرمجة الوظيفية
- **csharp-pro**: تطوير C# الحديث، أطر .NET، أنماط المؤسسات
- **scala-pro**: سكالَا للمؤسسات مع البرمجة الوظيفية، Apache Pekko/Akka actors، Apache Spark، ZIO/Cats Effect، هندسات تفاعلية
- **unity-developer**: تطوير الألعاب بـ Unity، البرمجة بلغة C#، تحسين الأداء
- **minecraft-bukkit-pro**: تطوير إضافات Minecraft، أنظمة الأحداث، ميزات السيرفر
- **ios-developer**: تطوير iOS الأصلي باستخدام Swift/SwiftUI
- **sql-pro**: استعلامات قواعد البيانات، تصميم المخططات، تحسين الاستعلامات
- **mobile-developer**: تطوير React Native/Flutter

### 🛠️ العمليات والصيانة
- **devops-troubleshooter**: مشاكل الإنتاج، مشاكل النشر
- **incident-responder**: الأعطال الحرجة التي تتطلب استجابة فورية
- **database-optimizer**: أداء الاستعلامات، استراتيجيات الفهرسة
- **database-admin**: استراتيجيات النسخ الاحتياطي، التكرار، إدارة المستخدمين، استعادة الكوارث
- **terraform-specialist**: البنية التحتية ككود، وحدات Terraform، إدارة الحالة
- **network-engineer**: اتصال الشبكة، موازنات التحميل، SSL/TLS، تصحيح أخطاء DNS

### 📊 التحليل والتحسين
- **performance-engineer**: اختناقات التطبيق، التحسين
- **security-auditor**: فحص الثغرات، فحوصات الامتثال
- **data-scientist**: تحليل البيانات، الاستنتاجات، التقارير
- **mlops-engineer**: بنية تحتية لتعلم الآلة، تتبع التجارب، سجلات النماذج، أتمتة خطوط الإنتاج

### 🧪 ضمان الجودة
- **مراجع الكود**: جودة الكود، أمان التكوين، موثوقية الإنتاج
- **مُؤتمت الاختبار**: استراتيجية الاختبار، إنشاء مجموعة الاختبارات
- **مصحح الأخطاء**: تحقيق في الأخطاء، حل المشكلات
- **محقق الأخطاء**: تحليل السجلات، التعرف على أنماط الأخطاء، تحليل السبب الجذري
- **أخصائي البحث**: بحث عميق في الويب، تحليل المنافسين، التحقق من الحقائق

### 📚 التوثيق
- **موثق API**: مواصفات OpenAPI/Swagger، توثيق واجهات البرمجة
- **مهندس التوثيق**: توثيق تقني شامل، أدلة هيكلية، كتيبات النظام
- **بناء المرجع**: مراجع API شاملة، أدلة التكوين، توثيق المعاملات
- **مهندس الدروس**: دروس خطوة بخطوة، مسارات تعلم، محتوى تعليمي

### 💼 الأعمال والاستراتيجية
- **محلل الأعمال**: مؤشرات الأداء، نماذج الإيرادات، توقعات النمو، مقاييس المستثمرين

- **مؤتمت المبيعات**: رسائل بريد باردة، متابعات، مقترحات، رعاية العملاء المحتملين
- **دعم العملاء**: تذاكر الدعم، الأسئلة الشائعة، مستندات المساعدة، استكشاف الأخطاء وإصلاحها
- **مستشار قانوني** - صياغة سياسات الخصوصية، شروط الخدمة، إخلاء المسؤولية، والإشعارات القانونية
