# Koleksi Subagen Claude Code

Koleksi lengkap subagen AI khusus untuk [Claude Code](https://docs.anthropic.com/en/docs/claude-code), dirancang untuk meningkatkan alur kerja pengembangan dengan keahlian spesifik bidang.

## Ikhtisar

Repositori ini berisi 58 subagen khusus yang memperluas kemampuan Claude Code. Setiap subagen merupakan ahli dalam bidang tertentu, dipanggil secara otomatis berdasarkan konteks atau secara eksplisit saat dibutuhkan. Semua agen dikonfigurasi dengan model Claude tertentu sesuai kompleksitas tugas untuk performa optimal dan efisiensi biaya.

## Format Subagen

Setiap subagen mengikuti struktur berikut:
```markdown
---
name: subagent-name
description: When this subagent should be invoked
model: haiku  # Optional - specify which model to use (haiku/sonnet/opus)
tools: tool1, tool2  # Optional - defaults to all tools
---

System prompt defining the subagent's role and capabilities
```

### Konfigurasi Model

Mulai Claude Code v1.0.64, subagen dapat menentukan model Claude yang akan mereka gunakan. Ini memungkinkan delegasi tugas yang hemat biaya berdasarkan kompleksitas:

- **Kompleksitas Rendah (Haiku)**: Tugas sederhana seperti analisis data dasar, pembuatan dokumentasi, dan respons standar
- **Kompleksitas Sedang (Sonnet)**: Tugas pengembangan, review kode, pengujian, dan pekerjaan rekayasa standar  
- **Kompleksitas Tinggi (Opus)**: Tugas kritis seperti audit keamanan, review arsitektur, respons insiden, dan rekayasa AI/ML

Model yang tersedia (menggunakan penamaan sederhana per Claude Code v1.0.64):
- `haiku` - Cepat dan hemat biaya untuk tugas sederhana
- `sonnet` - Performa seimbang untuk sebagian besar pekerjaan pengembangan
- `opus` - Paling mampu untuk analisis kompleks dan tugas kritis

Jika tidak ada model yang ditentukan, subagen akan menggunakan model default sistem.

## Pola Orkestrasi Agen

Claude Code secara otomatis mengoordinasikan agen menggunakan pola umum berikut:

### Alur Kerja Berurutan
```
User Request ‚Üí Agent A ‚Üí Agent B ‚Üí Agent C ‚Üí Result

Example: "Build a new API feature"
backend-architect ‚Üí frontend-developer ‚Üí test-automator ‚Üí security-auditor
```

### Parallel Execution
```
User Request ‚Üí Agent A + Agent B (simultaneously) ‚Üí Merge Results

Example: "Optimize application performance" 
performance-engineer + database-optimizer ‚Üí Combined recommendations
```

### Percabangan Bersyarat
```
User Request ‚Üí Analysis ‚Üí Route to appropriate specialist

Example: "Fix this bug"
debugger (analyzes) ‚Üí Routes to: backend-architect OR frontend-developer OR devops-troubleshooter
```

### Tinjauan & Validasi
```
Primary Agent ‚Üí Review Agent ‚Üí Final Result

Example: "Implement payment processing"
payment-integration ‚Üí security-auditor ‚Üí Validated implementation
```

## Pemecahan Masalah  

### Masalah Umum  

**Agen tidak diaktifkan secara otomatis:**  
- Pastikan permintaan Anda jelas menunjukkan domainnya (misal, "masalah performa" ‚Üí performance-engineer)  
- Spesifikkan tipe tugasnya (misal, "review kode" ‚Üí code-reviewer)  

**Pemilihan agen tidak terduga:**  
- Berikan lebih banyak konteks tentang teknologi dan persyaratan Anda  
- Gunakan pemanggilan eksplisit jika Anda memerlukan agen tertentu  

**Beberapa agen menghasilkan saran yang bertentangan:**  
- Ini normal - spesialis yang berbeda dapat memiliki prioritas berbeda  
- Minta klarifikasi: "Rekonsiliasi rekomendasi dari security-auditor dan performance-engineer"  

**Agen tampak kurang konteks:**  
- Berikan informasi latar belakang dalam permintaan Anda  
- Referensi percakapan sebelumnya atau pola yang sudah ada  

### Mendapatkan Bantuan  

Jika agen tidak berfungsi seperti yang diharapkan:  
1. Periksa deskripsi agen di file masing-masing  
2. Coba gunakan bahasa yang lebih spesifik dalam permintaan Anda  
3. Gunakan pemanggilan eksplisit untuk menguji agen tertentu  
4. Berikan lebih banyak konteks tentang proyek dan tujuan Anda
