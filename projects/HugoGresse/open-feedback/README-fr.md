**Open Feedback** est une plateforme SaaS ouverte pour les organisateurs et intervenants afin de recueillir les retours des utilisateurs. Elle se connecte à une des 3 sources de données : base de données OpenFeedback, hoverboard-v2, ou votre propre API, pour récupérer toutes les sessions et intervenants. Les utilisateurs anonymes peuvent donner des retours constructifs sur les sessions, et les administrateurs peuvent gérer le formulaire de vote, les commentaires, les langues, les sessions et intervenants directement via l’interface graphique. L’organisation dispose également d’une gestion centralisée des utilisateurs et d’un formulaire de vote commun.

> Démo : https://openfeedback.io/eaJnyMXD3oNfhrrnBYDT

## Fonctionnalités

-   Générer une URL unique et publique en lecture pour un événement
-   Lister toutes les sessions, rechercher par tag, intervenants, pistes
-   Les utilisateurs donnent un retour (pas une note) sur n’importe quelle session de manière anonyme (utilise la connexion anonyme Firebase)
-   Les formulaires de vote sont 100% personnalisables et peuvent être traduits en toutes langues
-   Afficher les retours des utilisateurs, avec option de retour en temps réel
-   Statistiques complètes du tableau de bord et interface administrateur
-   Modération des commentaires
-   Gestion des éditeurs administrateurs
-   Organisation : gérer les utilisateurs, les droits, le formulaire de vote et les logos

## SDK Communauté
OpenFeedback ne dispose pas de SDK officiels pour intégrer OpenFeedback dans une application native existante, mais la communauté a créé ces bibliothèques formidables :  
- **SDK Kotlin** _par [Gérard Paligo](https://github.com/GerardPaligot/) et [Martin Bonnin](https://github.com/martinbonnin/)_: https://github.com/paug/openfeedback-android-sdk
