## 🎯 Por qué Chain-Fox

- Las auditorías de seguridad son **caras y consumen mucho tiempo**.  
- Las herramientas existentes están **fragmentadas y son difíciles de integrar**.  
- Muchos equipos omiten auditorías por completo — dejando vulnerabilidades críticas en producción.

**Chain-Fox cambia eso.**  
Ofrecemos detección de seguridad automatizada y multilenguaje que es fácil de integrar en tu flujo de trabajo — empoderando a los desarrolladores para entregar código más seguro sin los altos costos de auditorías manuales.

## 🛣️ Hoja de ruta

Chain-Fox está evolucionando de auditorías solo de código a análisis completo de riesgos Web3 usando Skills y agentes.  
Para más detalles, ver [ROADMAP.md](https://raw.githubusercontent.com/Chain-Fox/Chain-Fox/main/./ROADMAP.md)   

### Corto Plazo (Mes 1)
- Detección de rug pull (permisos + comportamiento on-chain)
- Chequeos de riesgo en sitios web Web3
- Auditoría de contratos basada en habilidades  
- Hacer que los verificadores de código sean agenticos y conscientes de habilidades  
- Pruebas beta y recopilación de comentarios  

### Mediano Plazo (Mes 2)  
- Mejorar las señales y reducir falsos positivos/negativos  
- Añadir más habilidades y verificadores de código especializados  
- Rastrear patrones históricos en cadena  
- Documentación API e integración  

### Largo Plazo (Trimestre)  
- Sistema de habilidades multi-agente  
- Capa de inteligencia de amenazas  
- Integración CI/CD y alertas  
- Extensión de navegador (solo lectura)  
- Habilidades de código abierto y contribuciones comunitarias

## 🔍 Verificadores Soportados  

Reunimos las herramientas líderes en los ecosistemas:  

| Verificador  | Lenguaje   | Tipo      | Detecta |
| ------------ | ---------- | --------- | ------- |
| lockbud      | Rust       | estático  | Errores de memoria y concurrencia |
| rudra        | Rust       | estático  | Seguridad de memoria, variancia, errores de duración |
| RAPx         | Rust       | estático  | Uso después de liberar, fugas de memoria |
| AtomVChecker | Rust       | estático  | Errores atómicos de concurrencia |
| Cocoon       | Rust       | estático  | Fugas de secreto |
| MIRAI        | Rust       | estático  | Problemas de pánico y corrección |
| ERASan       | Rust       | dinámico  | Errores de acceso a memoria |
| shuttle      | Rust       | dinámico  | Errores de concurrencia |
| kani         | Rust       | verificador | Seguridad, aserciones, pánicos |
| GCatch       | Go         | estático  | Errores de concurrencia |
| GFuzz        | Go         | dinámico  | Errores de concurrencia |
| cppcheck     | C/C++      | estático  | Errores comunes en C/C++ |
| slither      | Solidity   | estático  | Errores en Solidity |
| PeCatch      | Solidity   | estático  | Ineficiencias en costos de gas |
