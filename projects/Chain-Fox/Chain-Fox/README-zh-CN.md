## 🎯 为什么选择 Chain-Fox

- 安全审计既**昂贵又耗时**。  
- 现有工具**零散且难以集成**。  
- 许多团队完全跳过审计——导致关键漏洞留在生产环境中。

**Chain-Fox 改变了这一切。**  
我们提供自动化、多语言的安全检测，易于集成到您的工作流程中——让开发者无需高额人工审计成本也能发布更安全的代码。

## 🔍 支持的检查器  

我们汇集了跨生态系统的领先工具：  

| 检查器       | 语言       | 类型      | 检测内容                |  
| ------------ | ---------- | --------- | ----------------------- |  
| lockbud      | Rust       | 静态      | 内存和并发错误          |  
| rudra        | Rust       | 静态      | 内存安全、变异性、生命周期错误 |  
| RAPx         | Rust       | 静态      | 使用后释放、内存泄漏    |  
| AtomVChecker | Rust       | 静态      | 原子并发错误            |  
| Cocoon       | Rust       | 静态      | 保密泄露                |  
| MIRAI        | Rust       | 静态      | 异常与正确性问题        |
| ERASan       | Rust       | 动态      | 内存访问错误 |
| shuttle      | Rust       | 动态      | 并发错误 |
| kani         | Rust       | 验证器    | 安全性、断言、恐慌 |
| GCatch       | Go         | 静态      | 并发错误 |
| GFuzz        | Go         | 动态      | 并发错误 |
| cppcheck     | C/C++      | 静态      | 常见C/C++错误 |
| slither      | Solidity   | 静态      | Solidity错误 |
| PeCatch      | Solidity   | 静态      | 燃气费低效 |

👉 请参见我们的[Awesome Rust Checker](https://github.com/Chain-Fox/Awesome-Rust-Checker)列表，了解即将添加的工具。
