## 🎯 Pourquoi Chain-Fox

- Les audits de sécurité sont **coûteux et chronophages**.  
- Les outils existants sont **fragmentés et difficiles à intégrer**.  
- De nombreuses équipes sautent complètement les audits — laissant des vulnérabilités critiques en production.

**Chain-Fox change cela.**  
Nous offrons une détection de sécurité automatisée, multilingue, facile à intégrer dans votre flux de travail — permettant aux développeurs de livrer un code plus sûr sans les coûts élevés des audits manuels.

## 🛣️ Feuille de route

Chain-Fox évolue de l’audit uniquement code vers une analyse complète des risques Web3 utilisant Skills et agents.  
Pour plus de détails, voir [ROADMAP.md](https://raw.githubusercontent.com/Chain-Fox/Chain-Fox/main/./ROADMAP.md)   

### Court terme (Mois 1)
- Détection de rug pull (permissions + comportement on-chain)
- Vérifications des risques des sites Web3
- Audit de contrat basé sur les compétences  
- Rendre les vérificateurs de code agentiques et conscients des compétences  
- Tests bêta et collecte de retours  

### Moyen terme (Mois 2)  
- Améliorer les signaux et réduire les faux positifs/négatifs  
- Ajouter plus de compétences et de vérificateurs de code spécialisés  
- Suivre les motifs historiques on-chain  
- Documentation API et intégration  

### Long terme (Trimestre)  
- Système multi-agent de compétences  
- Couche de renseignement sur les menaces  
- Intégration CI/CD et alertes  
- Extension de navigateur (lecture seule)  
- Compétences open-source et contributions communautaires

## 🔍 Vérificateurs Pris en Charge  

Nous réunissons les meilleurs outils de différents écosystèmes :  

| Vérificateur | Langage   | Type      | Détecte                          |  
| ------------ | --------- | --------- | ------------------------------- |  
| lockbud      | Rust      | statique  | Bugs mémoire et de concurrence  |  
| rudra        | Rust      | statique  | Sécurité mémoire, variance, bugs de durée de vie |  
| RAPx         | Rust      | statique  | Use-after-free, fuites mémoire  |  
| AtomVChecker | Rust      | statique  | Bugs de concurrence atomique    |  
| Cocoon       | Rust      | statique  | Fuites de secret                |  
| MIRAI        | Rust      | statique  | Paniques et problèmes de correction |
| ERASan       | Rust       | dynamique | Bugs d'accès mémoire |
| shuttle      | Rust       | dynamique | Bugs de concurrence |
| kani         | Rust       | vérificateur | Sécurité, assertions, panics |
| GCatch       | Go         | statique  | Bugs de concurrence |
| GFuzz        | Go         | dynamique | Bugs de concurrence |
| cppcheck     | C/C++      | statique  | Bugs communs en C/C++ |
| slither      | Solidity   | statique  | Bugs Solidity |
| PeCatch      | Solidity   | statique  | Inefficacités des frais de gaz |

👉 Consultez notre liste [Awesome Rust Checker](https://github.com/Chain-Fox/Awesome-Rust-Checker) pour les ajouts à venir.
