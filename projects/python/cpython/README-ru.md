Инструкции по сборке
--------------------

На Unix, Linux, BSD, macOS и Cygwin::

    ./configure
    make
    make test
    sudo make install

Это установит Python как ``python3``.

Вы можете передать множество опций скрипту configure; выполните ``./configure --help``,
чтобы узнать больше. На файловых системах macOS с нечувствительностью к регистру и на Cygwin
исполняемый файл называется ``python.exe``; в остальных случаях — просто ``python``.

Для сборки полной установки Python требуется использование различных
дополнительных сторонних библиотек, в зависимости от вашей платформы сборки и
опций configure. Не все модули стандартной библиотеки можно собрать или
использовать на всех платформах. Обратитесь к разделу
`Установка зависимостей <https://devguide.python.org/getting-started/setup-building.html#build-dependencies>`_
в `Руководстве разработчика`_ для актуальной подробной информации о
зависимостях для различных дистрибутивов Linux и macOS.

На macOS есть дополнительные опции configure и сборки, связанные
с фреймворком macOS и универсальными сборками. См. `Mac/README.rst
<https://github.com/python/cpython/blob/main/Mac/README.rst>`_.

В Windows смотрите `PCbuild/readme.txt
<https://github.com/python/cpython/blob/main/PCbuild/readme.txt>`_.

Для сборки установщика Windows смотрите `Tools/msi/README.txt
<https://github.com/python/cpython/blob/main/Tools/msi/README.txt>`_.

Если хотите, вы можете создать подкаталог и вызвать configure оттуда.
Например::

    mkdir debug
    cd debug
    ../configure --with-pydebug
    make
    make test

(Это завершится неудачей, если вы *также* собирали в корневом каталоге. Сначала выполните
``make clean`` в корневом каталоге.)

Чтобы получить оптимизированную сборку Python, используйте ``configure --enable-optimizations``
перед запуском ``make``. Это настроит цели make по умолчанию для включения
оптимизации с профилированием (PGO) и может автоматически включить оптимизацию времени компоновки (LTO)
на некоторых платформах. Подробнее смотрите разделы ниже.

Оптимизация с профилированием (PGO)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

PGO использует преимущества последних версий компиляторов GCC или Clang. Если используется,
через ``configure --enable-optimizations`` или вручную через
``make profile-opt`` независимо от флагов configure, процесс оптимизированной сборки
выполнит следующие шаги:

Весь каталог Python очищается от временных файлов, которые могли остаться
после предыдущей компиляции.

Собирается инструментированная версия интерпретатора с использованием подходящих флагов компилятора для каждого варианта. Обратите внимание, что это только промежуточный этап. 
Бинарник, полученный на этом этапе, не подходит для реальных рабочих нагрузок, так как содержит встроенные инструкции профилирования.

После сборки инструментированного интерпретатора Makefile выполнит тренировочную нагрузку.
Это необходимо для профилирования выполнения интерпретатора.
Также обратите внимание, что любой вывод, как stdout, так и stderr, на этом этапе подавляется.

На заключительном этапе собирается фактический интерпретатор с использованием информации,
собранной с инструментированной версии. В результате получится бинарник Python,
оптимизированный для распространения или промышленной установки.

Оптимизация времени компоновки (LTO)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Включается через флаг configure ``--with-lto``. LTO использует возможности
современных инструментальных цепочек компиляторов для оптимизации за пределами
границ отдельных файлов ``.o`` при сборке финальных исполняемых файлов или
разделяемых библиотек для дополнительного прироста производительности.

Тестирование
------------

Для тестирования интерпретатора выполните ``make test`` в корневом каталоге. 
Набор тестов выдаёт некоторый вывод. Обычно можно игнорировать сообщения о
пропущенных тестах из-за необязательных функций, которые не могут быть импортированы. 
Если появляется сообщение о неудачном тесте, трассировке или создаётся core dump,
значит что-то не так.

По умолчанию тесты защищены от чрезмерного использования ресурсов, таких как диск и память.
Чтобы включить эти тесты, выполните ``make buildbottest``.

Если какие-либо тесты завершились неудачей, вы можете повторно запустить их в подробном режиме.
Например, если не прошли ``test_os`` и ``test_gdb``, выполните::

    make test TESTOPTS="-v test_os test_gdb"

Если сбой сохраняется и кажется, что проблема связана с Python, а не с вашей средой,
вы можете `отправить отчёт об ошибке
<https://github.com/python/cpython/issues>`_ и приложить соответствующий вывод этой команды для демонстрации проблемы.

См. `Запуск и написание тестов <https://devguide.python.org/testing/run-write-tests.html>`_
для получения информации о запуске тестов.

Установка нескольких версий
----------------------------

В Unix и Mac, если вы планируете установить несколько версий Python
с одним и тем же префиксом установки (аргумент ``--prefix`` скрипта configure),
вы должны позаботиться о том, чтобы ваш основной исполняемый файл python не был
перезаписан при установке другой версии. Все файлы и каталоги, установленные с помощью
``make altinstall``, содержат основные и второстепенные номера версий, поэтому могут сосуществовать.
``make install`` также создаёт ``${prefix}/bin/python3``, который ссылается на ``${prefix}/bin/python3.X``.
Если вы планируете установить несколько версий с одним префиксом, вы должны решить, какая версия (если вообще)
будет вашей "основной" версией. Установите эту версию с помощью
``make install``.  Установите все остальные версии с помощью ``make altinstall``.

Например, если вы хотите установить Python 2.7, 3.6 и 3.15, где 3.15 будет
основной версией, выполните ``make install`` в каталоге сборки 3.15
и ``make altinstall`` в остальных.
