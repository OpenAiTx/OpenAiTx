# Svelte Preprocess React

Usa componentes React sin problemas dentro de una aplicación Svelte

Soporta:

- Anidamiento (Slot y Children)
- Contextos
- SSR
- Hooks ([useStore](https://raw.githubusercontent.com/bfanger/svelte-preprocess-react/main/./docs/useStore.md) y [hooks](https://raw.githubusercontent.com/bfanger/svelte-preprocess-react/main/./docs/hooks.md))

Este proyecto fue presentado en el [Svelte London - Meetup de noviembre 2022](https://www.youtube.com/live/DXQl1G54DJY?feature=share&t=2569)

> "Adoptar, extender y extinguir"

Este preprocesador está pensado como solución para usar componentes React de terceros o para migrar una base de código React existente.

## Usando React dentro de componentes Svelte

Dentro de la plantilla Svelte antepone el nombre del componente con el prefijo `react.`

En lugar de `<Button>`, escribirías `<react.Button>`

Usa librerías del ecosistema React, react-youtube por ejemplo:

```svelte
<script>
  import YouTube from "react-youtube";

  const react = sveltify({ YouTube }); // Optional step, but adds type-safety
</script>

<react.YouTube videoId="AdNJ3fydeao" />
```

El fragmento anterior generaría:

```svelte
<!-- Generated by svelte-preprocess-react -->
<script>
  import ReactDOM from "react-dom/client";
  import { createPortal } from "react-dom";
  import { renderToString } from "react-dom/server";
  import { sveltify } from "svelte-preprocess-react";
  import YouTube from "react-youtube";

  const react = sveltify(
    { YouTube },
    { createPortal, ReactDOM, renderToString },
  );
</script>

<react.YouTube videoId="AdNJ3fydeao" />
```

## Configuración / Instalación

```sh
npm install --save-dev svelte-preprocess-react react react-dom
```

Agrega `preprocessReact` a tu svelte.config.js:

```js
// svelte.config.js
import preprocessReact from "svelte-preprocess-react/preprocessReact";

export default {
  preprocess: preprocessReact(),
};
```

Cuando utilice otros procesadores como [@sveltejs/vite-plugin-svelte](https://github.com/sveltejs/vite-plugin-svelte/blob/main/docs/preprocess.md) o [svelte-preprocess](https://github.com/sveltejs/svelte-preprocess) agregue el preprocesador preprocessReact como el último procesador:

```js
// svelte.config.js
import { vitePreprocess } from "@sveltejs/vite-plugin-svelte";
import preprocessReact from "svelte-preprocess-react/preprocessReact";

export default {
  preprocess: [vitePreprocess(), preprocessReact()],
};
```

## Usando Svelte dentro de componentes React

Una vez que hayas convertido un componente React a Svelte, querrás eliminar ese componente React, pero si otros componentes React dependían de ese componente, puedes usar `reactify` para usar el nuevo componente Svelte como un componente React.

```jsx
import { reactify } from "svelte-preprocess-react";
import Button from "../components/Button.svelte";

const svelte = reactify({ Button });

function MyComponent() {
  return (
    <svelte.Button onClick={() => console.log("clicked")}>
      Click me
    </svelte.Button>
  );
}
```

## Usar múltiples frameworks es una mala idea

Usar múltiples frameworks de frontend añade sobrecarga, tanto en la experiencia del usuario como en la del desarrollador.

- Tamaño de descarga incrementado
- Más lento (cada límite de framework añade sobrecarga)
- Cambio de contexto, mantener las particularidades de Svelte y React en tu cabeza ralentiza el desarrollo

Cuando uses componentes React de terceros, busca alternativas en Svelte, o publica las tuyas propias.

Cuando se use como herramienta de migración (puede usarse para migrar _desde_ o _hacia_ React),
el objetivo debería ser dejar de escribir nuevos componentes React y convertir los componentes React existentes a componentes Svelte.
Una vez convertidos todos los componentes, este preprocesador debería desinstalarse.
