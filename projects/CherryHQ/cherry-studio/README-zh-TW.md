# 🍒 Cherry Studio

Cherry Studio 是一款支援多種 LLM 供應商的桌面客戶端，適用於 Windows、Mac 和 Linux。

👏 加入 [Telegram 群組](https://t.me/CherryStudioAI)｜[Discord](https://discord.gg/wez8HtpxqQ) | [QQ 群(575014769)](https://qm.qq.com/q/lo0D4qVZKi)

# 🌟 主要功能

1. **多元 LLM 供應商支援**：

- ☁️ 主流 LLM 雲端服務：OpenAI、Gemini、Anthropic 等
- 🔗 AI 網頁服務整合：Claude、Peplexity、Poe 及其他
- 💻 本地模型支援 Ollama、LM Studio

2. **AI 助理與對話**：

- 📚 300+ 預設配置的 AI 助理
- 🤖 自訂助理建立
- 💬 多模型同時對話

3. **文件與資料處理**：

- 📄 支援文字、圖片、Office、PDF 等多種格式
- ☁️ WebDAV 檔案管理與備份
- 📊 Mermaid 圖表視覺化
- 💻 程式碼語法高亮

4. **實用工具整合**：

- 🔍 全域搜尋功能
- 📝 主題管理系統
- 🔤 AI 驅動翻譯
- 🎯 拖放排序
- 🔌 小程式支援
- ⚙️ MCP（模型上下文協議）伺服器

5. **增強用戶體驗**：

- 🖥️ 跨平台支援 Windows、Mac、Linux
- 📦 開箱即用，無需環境配置
- 🎨 明暗主題與透明視窗
- 📝 完整 Markdown 渲染
- 🤲 內容輕鬆分享

# 🤝 參與貢獻

我們歡迎對 Cherry Studio 的貢獻！您可以透過以下方式參與：

1. **貢獻程式碼**：開發新功能或優化現有程式碼。
2. **修復錯誤**：提交您發現的錯誤修正。
3. **維護議題**：協助管理 GitHub 議題。
4. **產品設計**：參與設計討論。
5. **撰寫文件**：改進用戶手冊與指南。
6. **社群互動**：參與討論並協助用戶。
7. **推廣應用**：宣傳 Cherry Studio。

請參考 [分支策略](https://raw.githubusercontent.com/CherryHQ/cherry-studio/main/docs/branching-strategy-en.md) 以瞭解貢獻指南

## 開始貢獻

1. **Fork 儲存庫**：Fork 並 Clone 到本地端。
2. **建立分支**：針對您的更改建立分支。
3. **提交更改**：Commit 並 Push 您的更改。
4. **建立 Pull Request**：描述您的更改及原因。

更多詳細說明請參閱我們的 [貢獻指南](https://raw.githubusercontent.com/CherryHQ/cherry-studio/main/./CONTRIBUTING.md)。

感謝您的支持與貢獻！
