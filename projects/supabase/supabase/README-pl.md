# Supabase

[Supabase](https://supabase.com) to otwartoźródłowa alternatywa dla Firebase. Budujemy funkcje Firebase wykorzystując narzędzia open source klasy korporacyjnej.

- [x] Hostowana baza danych Postgres. [Dokumentacja](https://supabase.com/docs/guides/database)
- [x] Uwierzytelnianie i autoryzacja. [Dokumentacja](https://supabase.com/docs/guides/auth)
- [x] Automatycznie generowane API.
  - [x] REST. [Dokumentacja](https://supabase.com/docs/guides/api)
  - [x] GraphQL. [Dokumentacja](https://supabase.com/docs/guides/graphql)
  - [x] Subskrypcje w czasie rzeczywistym. [Dokumentacja](https://supabase.com/docs/guides/realtime)
- [x] Funkcje.
  - [x] Funkcje bazy danych. [Dokumentacja](https://supabase.com/docs/guides/database/functions)
  - [x] Edge Functions [Dokumentacja](https://supabase.com/docs/guides/functions)
- [x] Przechowywanie plików. [Dokumentacja](https://supabase.com/docs/guides/storage)
- [x] Narzędzia AI + Vector/Embeddings. [Dokumentacja](https://supabase.com/docs/guides/ai)
- [x] Panel administracyjny

## Jak to działa

Supabase to połączenie narzędzi open source. Budujemy funkcje Firebase, wykorzystując produkty open source klasy korporacyjnej. Jeśli narzędzia i społeczności istnieją, z licencją MIT, Apache 2 lub równoważną, używamy i wspieramy to narzędzie. Jeśli narzędzia nie ma, budujemy je i udostępniamy jako open source. Supabase nie jest 1:1 odpowiednikiem Firebase. Naszym celem jest zapewnienie deweloperom doświadczenia podobnego do Firebase przy użyciu narzędzi open source.

**Architektura**

Supabase to [platforma hostowana](https://supabase.com/dashboard). Możesz się zarejestrować i zacząć korzystać z Supabase bez instalowania czegokolwiek.
Możesz także [samodzielnie hostować](https://supabase.com/docs/guides/hosting/overview) oraz [rozwijać lokalnie](https://supabase.com/docs/guides/local-development).

![Architektura](apps/docs/public/img/supabase-architecture.svg)

- [Postgres](https://www.postgresql.org/) to obiektowo-relacyjny system bazodanowy z ponad 30-letnim aktywnym rozwojem, który zyskał silną reputację niezawodności, bogactwa funkcji i wydajności.
- [Realtime](https://github.com/supabase/realtime) to serwer Elixir, który pozwala nasłuchiwać inserty, aktualizacje i usuwania w PostgreSQL przez websockets. Realtime wykorzystuje wbudowaną replikację Postgresa do wykrywania zmian, konwertuje je do JSON i rozsyła przez websockets do autoryzowanych klientów.
- [PostgREST](http://postgrest.org/) to serwer www, który zamienia twoją bazę danych PostgreSQL bezpośrednio w RESTful API.
- [GoTrue](https://github.com/supabase/gotrue) to API uwierzytelniania oparte na JWT, upraszczające rejestracje, logowania i zarządzanie sesjami w twoich aplikacjach.
- [Storage](https://github.com/supabase/storage-api) RESTful API do zarządzania plikami w S3, z uprawnieniami obsługiwanymi przez Postgres.
- [pg_graphql](http://github.com/supabase/pg_graphql/) rozszerzenie PostgreSQL udostępniające API GraphQL.
- [postgres-meta](https://github.com/supabase/postgres-meta) RESTful API do zarządzania Postgres — pobierania tabel, dodawania ról, uruchamiania zapytań itp.
- [Kong](https://github.com/Kong/kong) to natywny dla chmury gateway API.
