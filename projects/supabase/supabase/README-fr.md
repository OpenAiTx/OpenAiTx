# Supabase

[Supabase](https://supabase.com) est une alternative open source à Firebase. Nous construisons les fonctionnalités de Firebase en utilisant des outils open source de qualité entreprise.

- [x] Base de données Postgres hébergée. [Docs](https://supabase.com/docs/guides/database)
- [x] Authentification et Autorisation. [Docs](https://supabase.com/docs/guides/auth)
- [x] APIs auto-générées.
  - [x] REST. [Docs](https://supabase.com/docs/guides/api)
  - [x] GraphQL. [Docs](https://supabase.com/docs/guides/graphql)
  - [x] Abonnements temps réel. [Docs](https://supabase.com/docs/guides/realtime)
- [x] Fonctions.
  - [x] Fonctions de base de données. [Docs](https://supabase.com/docs/guides/database/functions)
  - [x] Fonctions Edge [Docs](https://supabase.com/docs/guides/functions)

- [x] Boîte à outils IA + Vecteur/Embeddings. [Docs](https://supabase.com/docs/guides/ai)

## Comment ça marche

Supabase est une combinaison d'outils open source. Nous construisons les fonctionnalités de Firebase en utilisant des produits open source de niveau entreprise. Si les outils et communautés existent, avec une licence open source MIT, Apache 2 ou équivalente, nous utiliserons et soutiendrons cet outil. Si l'outil n'existe pas, nous le développons et le publions en open source nous-mêmes. Supabase n'est pas une réplique exacte de Firebase. Notre objectif est d'offrir aux développeurs une expérience similaire à Firebase en utilisant des outils open source.

**Architecture**

Supabase est une [plateforme hébergée](https://supabase.com/dashboard). Vous pouvez vous inscrire et commencer à utiliser Supabase sans rien installer.
Vous pouvez également [auto-héberger](https://supabase.com/docs/guides/hosting/overview) et [développer localement](https://supabase.com/docs/guides/local-development).

![Architecture](apps/docs/public/img/supabase-architecture.svg)

- [Postgres](https://www.postgresql.org/) est un système de base de données relationnelle objet avec plus de 30 ans de développement actif, ce qui lui a valu une solide réputation en termes de fiabilité, de robustesse des fonctionnalités et de performance.
- [Realtime](https://github.com/supabase/realtime) est un serveur Elixir qui vous permet d'écouter les insertions, mises à jour et suppressions PostgreSQL via websockets. Realtime interroge la fonctionnalité de réplication intégrée de Postgres pour les changements de la base de données, convertit les changements en JSON, puis diffuse le JSON via websockets aux clients autorisés.
- [PostgREST](http://postgrest.org/) est un serveur web qui transforme directement votre base de données PostgreSQL en une API RESTful.
- [GoTrue](https://github.com/supabase/gotrue) est une API d'authentification basée sur JWT qui simplifie l'inscription des utilisateurs, les connexions et la gestion des sessions dans vos applications.
- [Storage](https://github.com/supabase/storage-api) une API RESTful pour gérer les fichiers dans S3, avec Postgres gérant les permissions.
- [pg_graphql](http://github.com/supabase/pg_graphql/) une extension PostgreSQL qui expose une API GraphQL.
- [postgres-meta](https://github.com/supabase/postgres-meta) est une API RESTful pour gérer votre Postgres, permettant de récupérer des tables, d'ajouter des rôles, d'exécuter des requêtes, etc.
- [Kong](https://github.com/Kong/kong) est une passerelle d'API cloud-native.

#### Bibliothèques clientes

Notre approche pour les bibliothèques clientes est modulaire. Chaque sous-bibliothèque est une implémentation autonome pour un seul système externe. C'est l'une des façons dont nous soutenons les outils existants.
