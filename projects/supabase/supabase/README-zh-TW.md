# Supabase

[Supabase](https://supabase.com) 是一個開源的 Firebase 替代方案。我們正在使用企業級的開源工具構建 Firebase 的各項功能。

- [x] 託管的 Postgres 資料庫。 [文件](https://supabase.com/docs/guides/database)
- [x] 認證與授權。 [文件](https://supabase.com/docs/guides/auth)
- [x] 自動產生的 API。
  - [x] REST。 [文件](https://supabase.com/docs/guides/api)
  - [x] GraphQL。 [文件](https://supabase.com/docs/guides/graphql)
  - [x] 即時訂閱。 [文件](https://supabase.com/docs/guides/realtime)
- [x] 函式。
  - [x] 資料庫函式。 [文件](https://supabase.com/docs/guides/database/functions)
  - [x] Edge 函式 [文件](https://supabase.com/docs/guides/functions)
- [x] 檔案儲存。 [文件](https://supabase.com/docs/guides/storage)
- [x] AI + 向量／嵌入工具包。 [文件](https://supabase.com/docs/guides/ai)
- [x] 控制台

## 運作原理

Supabase 是由多個開源工具組成。我們正在使用企業級、開源產品打造 Firebase 的各項功能。如果有現成的工具與社群，且採用 MIT、Apache 2 或同等開源授權，我們會直接使用並支援該工具。若沒有現成工具，我們會自行開發並開源。Supabase 並非 Firebase 的一對一對應，我們的目標是讓開發者能夠以開源工具體驗類似 Firebase 的開發者體驗。

**架構說明**

Supabase 是一個 [託管平台](https://supabase.com/dashboard)。您可以直接註冊並開始使用 Supabase，無需安裝任何軟體。
您也可以選擇 [自我託管](https://supabase.com/docs/guides/hosting/overview) 或 [本地開發](https://supabase.com/docs/guides/local-development)。

![Architecture](apps/docs/public/img/supabase-architecture.svg)

- [Postgres](https://www.postgresql.org/) 是一個具有 30 多年活躍開發歷史的物件關聯式資料庫系統，以其穩定性、功能強大和高效能著稱。
- [Realtime](https://github.com/supabase/realtime) 是一個 Elixir 伺服器，可透過 websockets 監聽 PostgreSQL 的新增、更新和刪除。Realtime 輪詢 Postgres 內建的複製功能以偵測資料庫變更，將變更轉換為 JSON，然後透過 websockets 廣播給授權用戶端。
- [PostgREST](http://postgrest.org/) 是一個網頁伺服器，能將您的 PostgreSQL 資料庫直接轉換為 RESTful API。
- [GoTrue](https://github.com/supabase/gotrue) 是一個基於 JWT 的認證 API，可簡化應用中的用戶註冊、登入和會話管理。
- [Storage](https://github.com/supabase/storage-api) 是一個 RESTful API，用於管理 S3 檔案，並由 Postgres 負責權限管理。
- [pg_graphql](http://github.com/supabase/pg_graphql/) 是一個 PostgreSQL 擴充套件，可提供 GraphQL API。
- [postgres-meta](https://github.com/supabase/postgres-meta) 是一個 RESTful API，用於管理您的 Postgres，支援查詢資料表、新增角色、執行查詢等功能。
- [Kong](https://github.com/Kong/kong) 是一個雲原生 API Gateway。
