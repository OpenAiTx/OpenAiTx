## 항상 관찰. 항상 준비.

대부분의 AI 도구는 요청을 기다립니다. CORE는 관찰합니다.

앱에 연결하면 모든 활동을 모니터링합니다. 클라이언트로부터 이메일이 도착하거나, GitHub 이슈가 할당되거나, Sentry 알림이 발생하거나, 회의가 끝나면 CORE는 이를 감지하고, 메모리와 설정된 스킬을 확인하여 직접 처리하거나 판단을 위해 보여줍니다. 당신이 직접 트리거하지 않아도 됩니다. CORE가 스스로 인지합니다.

Claude Code, Codex, 또는 Cursor에 CORE 플러그인을 설치하면 에이전트 대화도 관찰됩니다. 논의된 컨텍스트, 내린 결정, 작성된 코드 모두가 메모리 지식 그래프에 저장됩니다. CORE가 다음 작업을 집을 때 이미 지난 세션의 내용을 알고 있습니다.

CORE는 직접적으로 행동합니다. 이메일에 답장하고, Linear 이슈를 업데이트하며, GitHub PR을 작성하고, Slack 메시지를 보내고, 터미널 명령을 실행하며, 브라우저를 제어하거나, 어떤 인터페이스에서도 Claude Code 또는 Codex 세션을 생성할 수 있습니다. 공항에서 WhatsApp 메시지를 보내면 CORE가 코딩 세션을 실행하고 탑승 전에 PR을 열어둘 수 있습니다.

어디서 자동으로 동작할지, 어디서 당신의 호출을 기다릴지 직접 결정하세요. Sentry 알림은 자동으로 처리하지만 병합 전에는 항상 확인을 요청합니다. 코딩 세션 시작 전에는 계획을 승인해야 합니다. 이메일이 발송되기 전에는 반드시 확인을 요구합니다. 작업별, 앱별, 동작별로 자율성 수준을 직접 설정할 수 있습니다.

## 네 가지 접근 방식

챗봇에는 하나의 인터페이스가 있습니다. 운영체제(OS)에는 여러 개가 있습니다.

**음성.** Mac에서 Ctrl+Option을 누르고 해야 할 일을 말하세요. CORE는 백그라운드에서 실행되어 작업 흐름을 방해하지 않습니다.

**스크래치패드.** 오늘의 페이지를 열고 `[ ] 이슈 #47의 인증 버그 수정`이라고 작성하세요. CORE가 3분 이내에 이를 감지하고, 저장소와 메모리에서 컨텍스트를 불러와 계획 초안을 만듭니다.

**메시징.** WhatsApp, Slack, Telegram. 공항에서, 휴대폰에서, 침대에서 작업을 보낼 수 있습니다. 메시지의 출처와 관계없이 CORE는 전체 컨텍스트를 파악합니다.

**채팅.** 대시보드를 열고 직접 대화하세요. 위임 전에 대화를 주고받고 싶을 때 적합합니다.

하나의 AI, 네 가지 접점, 모두 동일한 메모리와 컨텍스트로 작동합니다.

## 나만의 AI로 만들기

이름을 정하세요. 말투를 선택하세요. 항상 따라야 할 규칙을 설정하세요.

내장된 다섯 가지 개성 중에서 선택하거나(TARS의 건조한 효율성, Alfred의 충직한 격식, Hudson의 따뜻한 실용성 등) 직접 개성을 작성할 수 있습니다. 이 개성은 AI가 수행하는 모든 작업, 작성하는 모든 계획, 대신 발송하는 모든 메시지에 반영됩니다.

음성 상호작용에서 사용할 목소리를 선택하세요. 그 순간부터, 모든 인터페이스에서 설정한 AI의 목소리와 행동이 적용됩니다.

## CORE 실제 사용 예시

### 말로 지시하고, PR로 돌아옵니다.

*Ctrl+Option을 누르고 말하세요:* "이슈 #312의 체크아웃 플로우에서 경쟁 조건을 수정해줘."

CORE가 이슈를 불러오고, 관련 커밋과 Slack 스레드를 가져오며, 계획을 초안하고 Claude Code 세션을 실행합니다. 당신은 diff만 확인하면 됩니다. 책상에 앉아 있을 필요가 없습니다.

### 밤에 작성하고, 아침에 검토하세요.

*스크래치패드:* `[ ] 오늘 밤 11시에 백로그 처리 시작`

CORE는 Linear, GitHub, 그리고 메모리에서 정보를 받아 우선순위를 정하고, 당신이 자는 동안 작업을 처리합니다. 매끄럽게 완료된 작업들은 검토를 기다리고 있습니다. 막힌 세션은 멈춰있는 탭이 아닌, 간결한 질문 하나로 돌아옵니다.

### 어디서든 메시지로 지시하세요.

*공항에서 WhatsApp:* "auth 리팩터링 배포해줘."

CORE는 이미 브랜치, 맥락, 당신의 선호도를 알고 있습니다. Railway에서 실행 중이며, 탑승 전에 세션을 시작합니다.

### 사고로 번지기 전에 알림을 조사하세요.

*Sentry가 새벽 2시에 알림을 보냅니다.* CORE가 조사하여 관련 트레이스와 과거 사고를 메모리에서 찾아내고, 수정안을 제안하며 Slack으로 알립니다: "이슈 #847, 수정안 제안됨, 검토 대기 중." 당신은 휴대폰으로 승인합니다.

### 당신의 한 주를 이미 아는 브리핑을 받으세요.

*매일 오전 8시의 반복 작업.* CORE는 이메일, GitHub, Linear, Slack에서 정보를 가져와 실제로 주목해야 할 것을 표면화하고, 불필요한 것은 건너뛰며, 후속 조치는 자동으로 작업으로 전환합니다.

## CORE의 구성 요소

| | |
|---|---|
| **메모리** | 모든 도구와 대화를 아우르는 시간적 지식 그래프. 선호, 결정, 목표, 지침까지 모두 기억하여 모든 작업이 맥락을 갖고 시작합니다. |
| **작업** | 단발성 또는 반복적 작업 단위에 사양, CORE의 계획, 실시간 상태, 전용 채팅 스레드를 포함합니다. 각 작업은 코딩, 브라우저, 터미널 세션을 생성할 수 있습니다. |
| **커넥터** | 하나의 MCP 엔드포인트를 통한 50개 이상의 앱 연결, 그리고 자동화용 웹훅 트리거. GitHub, Linear, Jira, Slack, Gmail, Calendar, Sentry, Notion, Todoist 등 지원. |
| **스킬** | 맥락에 따라 자동으로 실행되는 재사용 가능한 지침. 예: "수정안 계획 전에 항상 관련 Linear 이슈를 불러온다", "PR 오픈 전 테스트 실행", "작업 완료 시 Slack 요약 전송". 100개 이상 내장, 직접 작성도 가능. |
| **게이트웨이** | Claude Code, Codex, 브라우저 에이전트, 터미널 명령을 당신의 컴퓨터 또는 Docker, Railway에서 실행하여, 노트북이 닫혀도 CORE가 계속 작동합니다. |
| **모델 독립성** | 원하는 제공업체 선택: Anthropic, OpenAI, 또는 오픈 소스 모델. 전체 스택을 직접 호스팅하여 완전한 격리 가능. |

## 빠른 시작

오픈소스 및 자체 호스팅. 데이터는 귀하의 인프라 내에만 저장됩니다.

**원하는 경로를 선택하세요:**

| 하고 싶은 일 | 방법 |
|---|---|
| 내 컴퓨터에서 사용해보기 | 아래의 원스텝 설치 실행 (Docker 필요) |
| 서버 또는 VPS에 배포 | 원클릭 Railway 배포 |
| 맥 앱 사용 | [대기자 명단 신청하기](https://www.getcore.me/) |

**CORE 설치 및 시작:**

```bash
npm install -g @redplanethq/corebrain && corebrain setup
```
설정 마법사는 설치 디렉토리, AI 공급자, API 키, 그리고 챗 모델을 입력하도록 요청합니다. 비밀키를 생성하고, 스택을 시작하며, `http://localhost:3033`을 엽니다.

대부분의 로컬 설치는 Docker가 실행된 후 몇 분 정도 소요됩니다.

**또는 Railway에 배포하기:**

[![Railway에 배포하기](https://railway.app/button.svg)](https://railway.com/deploy/core)

**게이트웨이 연결하기**를 통해 CORE가 코딩 에이전트를 실행하고, 브라우저를 제어하며, 로컬 폴더에 접근할 수 있습니다:


```bash
corebrain login
corebrain gateway setup
```

**필수 조건:** Docker 20.10+, Docker Compose 2.20+, 4 vCPU / 8GB RAM

[전체 셀프 호스팅 가이드](https://docs.getcore.me/self-hosting/setup)

**설정 후 첫 번째 작업 (2분 소요):**

1. **스크래치패드**를 엽니다 (`http://localhost:3033`의 데일리 페이지)
2. `[ ] 내 열린 GitHub 이슈 요약` 또는 평소에 직접 하던 작업을 입력합니다
3. CORE가 3분 이내로 작업을 감지하고, 연결된 앱에서 컨텍스트를 수집하여 계획을 초안합니다
4. 계획을 승인하면 CORE가 실행하고 결과를 반환합니다

[첫 앱 연결하기](https://docs.getcore.me/connectors)
