CORE는 여러분의 디지털 브레인입니다. 통합된, 항상 동작하는 AI로서 여러분의 컨텍스트를 기억하고, AI 에이전트와 협업하며, 다양한 도구에서 행동합니다.

오픈소스. 자체 호스팅 가능. 데이터는 여러분의 소유, 여러분의 통제.

## 디지털 브레인의 조건

### 모든 것을 기억합니다

모든 대화, 결정, 선호, 관계가 시간 기반 지식 그래프의 구조화된 지식으로 저장됩니다. 단순 텍스트가 아니라 분류된 사실(선호, 결정, 목표, 지시)로 저장되어 필요할 때 정확히 드러납니다. 몇 주 전의 일이라도 물어보면 모든 컨텍스트와 함께 답을 얻을 수 있습니다.

[Memory에 대해 더 알아보기 →](https://docs.getcore.me/concepts/memory/overview)

### 행동을 취할 수 있습니다

GitHub 이슈를 생성하고, Slack 메시지를 보내며, 대화 중에 캘린더를 관리할 수 있습니다. 복잡한 다단계 워크플로도 가능: 코딩 에이전트에게 Sentry에서 새로운 오류를 확인하고, 각각에 대해 GitHub 이슈를 생성하며, git blame을 기반으로 적절한 엔지니어를 지정하고, 팀 Slack 채널에 요약을 게시하도록 한 번의 프롬프트에서 모두 처리할 수 있습니다. 50개 이상의 앱에서 200개 이상의 액션을 하나의 연결 지점으로 실행합니다.

[Toolkit에 대해 더 알아보기 →](https://docs.getcore.me/concepts/toolkit)

### 시스템 간 사고와 작업

당신의 도구들은 각각 고립되어 있습니다. Claude Code는 WhatsApp과 대화할 수 없습니다. Cursor는 캘린더를 확인할 수 없습니다. CORE의 에이전트가 이 벽을 허물고, 기억, 툴킷, 앱을 하나로 연결하는 오케스트레이터 역할을 합니다.

CORE Agent가 할 수 있는 일:

- **의도 이해**: 필요한 것을 분류하고 올바른 도구와 기억으로 연결
- **다른 에이전트 생성**: WhatsApp에서 Claude Code 세션이나 브라우저 세션을 시작하여 버그를 수정하거나 조사
- **다단계 워크플로 조정**: 기억에서 컨텍스트를 가져오고, 여러 앱에서 행동하며, 결과를 보고
- **어디에서든 작업**: WhatsApp, 이메일, Slack, 웹 대시보드에서 메시지 — 동일한 브레인, 동일한 컨텍스트

[CORE Agent에 대해 더 알아보기 →](https://docs.getcore.me/concepts/meta-agent)

### 능동적으로 행동할 수 있습니다

모든 것을 직접 요청할 필요는 없습니다. CORE는 이벤트를 모니터링합니다 — 새로운 이메일, GitHub 알림, 캘린더 변경 — 그리고 당신에 대해 알고 있는 것과 비교 평가합니다. 고객 이메일이 도착하면, CORE는 이미 관계를 알고 있고 적절한 답변을 초안 작성합니다. 밤에 Sentry 알림이 발생하면, CORE가 Claude Code 세션을 시작하여 조사합니다.

[Concepts에 대해 더 알아보기 →](https://docs.getcore.me/concepts/overview)

## 아키텍처

CORE는 이 네 가지 핵심 요소를 구현하는 세 개의 계층으로 구성되어 있습니다:

- **Memory**: 에피소드, 엔티티, 분류된 사실을 저장하는 시간 기반 지식 그래프. 모든 정보는 (선호, 결정, 지시, 목표 등)으로 분류되고 시간에 따라 연결됩니다. 검색이 키워드가 아닌 의도 기반이 되도록 합니다. [Docs →](https://docs.getcore.me/concepts/memory/overview)

- **Toolkit**: 모든 MCP 호환 에이전트를 위한 통합 액션 계층. 앱(GitHub, Linear, Slack, Gmail, Calendar 등)을 한 번 연결하면, 모든 연결된 AI 도구가 하나의 엔드포인트를 통해 200개 이상의 액션에 접근할 수 있습니다. [Docs →](https://docs.getcore.me/toolkit/overview)

- **CORE Agent**: 기억과 툴킷을 연결하는 오케스트레이터. 의도를 이해하고, 기억을 검색하며, 도구를 선택하고, 다른 에이전트를 생성하며, 트리거와 규칙에 따라 능동적으로 행동할 수 있습니다. [Docs →](https://docs.getcore.me/concepts/meta-agent)

## 에이전트 개발자를 위한 안내

AI 에이전트를 개발 중이신가요? 메모리와 통합 관리를 CORE에 맡기고, 에이전트의 로직에 집중하세요.

- **메모리 오프로드** — CORE의 시간 기반 지식 그래프를 에이전트의 장기 메모리로 사용하세요. 대화를 저장하고, 의도 기반 검색으로 맥락을 불러오며, 벡터 DB나 그래프를 직접 관리하지 않고도 에이전트가 지식을 축적할 수 있습니다.
- **통합 오프로드** — 앱을 CORE에 한 번만 연결하면, 에이전트는 모든 앱에 대한 MCP 도구를 사용할 수 있습니다. OAuth 플로우 개발, API 유지보수, 통합별 코드 작성이 필요 없습니다.
- **MCP 또는 API로 구축** — MCP(단일 엔드포인트)를 통해 에이전트를 CORE에 연결하거나 REST API를 직접 사용할 수 있습니다.

**예시 프로젝트**

- [**core-cli**](https://github.com/RedPlanetHQ/core-cli) — 메모리 및 Linear/GitHub 동기화가 가능한 작업 관리자 에이전트
- [**holo**](https://github.com/RedPlanetHQ/holo) — CORE 메모리를 개인 웹사이트 및 채팅으로 변환

[API 레퍼런스 →](https://docs.getcore.me/api-reference) · [SDK 문서 →](https://docs.getcore.me)

## 보안

CASA 2단계 인증 — Google OAuth 요건을 충족하는 제3자 감사를 완료했습니다.

- 암호화: TLS 1.3(전송 중) + AES-256(저장 시)
- 워크스페이스 기반 격리, 역할 기반 권한
- 귀하의 데이터는 AI 모델 학습에 절대 사용되지 않습니다
- 완전한 격리를 위한 셀프 호스팅 옵션
