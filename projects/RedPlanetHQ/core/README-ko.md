## 개요

**문제점**

개발자들은 AI 도구에 맥락을 반복해서 설명하는 데 시간을 낭비합니다. Claude에서 토큰 한도에 도달하면? 새로 시작하고 모든 것을 잃습니다. ChatGPT/Claude에서 Cursor로 바꾸면? 다시 맥락을 설명해야 합니다. 대화, 결정, 인사이트가 세션 사이에서 사라집니다.

**해결책** - **CORE** (**Contextual Observation & Recall Engine**)

CORE는 모든 AI 도구를 위한 오픈소스 통합, 지속적인 메모리 계층입니다. Cursor에서 Claude, ChatGPT, Claude Code까지 맥락이 따라옵니다. 하나의 지식 그래프가 누가, 언제, 왜 무엇을 말했는지 기억합니다. 한 번 연결하면 어디서든 기억됩니다. 맥락을 관리하지 말고 개발에 집중하세요.

## 🚀 시작하기
**5분 만에 통합 메모리 그래프 구축하기:**

1. [core.heysol.ai](https://core.heysol.ai)에서 **가입**하고 계정을 만드세요
2. **첫 번째 메모리 추가** - 자신에 대한 맥락을 공유하세요

    <img width="2088" height="1212" alt="first-memory" src="https://github.com/user-attachments/assets/ecfab88e-e91a-474d-9ef5-fc6c19b655a8" />


3. **메모리 그래프 시각화** - CORE가 사실 간의 연결을 자동으로 형성하는 과정을 확인하세요
5. **테스트 해보기** - 대화 섹션에서 "나에 대해 무엇을 알고 있나요?"라고 질문해보세요
6. 도구에 연결하기:
   - [Claude](https://docs.heysol.ai/providers/claude) & [Cursor](https://docs.heysol.ai/providers/cursor) - 맥락과 함께 코딩
   - [Claude Code CLI](https://docs.heysol.ai/providers/claude) & [Gemini CLI](https://docs.heysol.ai/providers/cursor) - 터미널 기반 메모리 코딩
   - [브라우저 확장 추가](https://docs.heysol.ai/providers/cursor) - 웹사이트 어디서든 메모리 사용
   - [Linear](https://docs.heysol.ai/providers/claude), [Slack](https://docs.heysol.ai/providers/cursor), [Github](https://docs.heysol.ai/providers/cursor) - 프로젝트 맥락 자동 추가

## 🧩 주요 기능
