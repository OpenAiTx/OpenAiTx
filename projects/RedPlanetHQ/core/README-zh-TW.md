## 隨時監控，隨時待命。

大多數 AI 工具都在等待指令，而 CORE 隨時監控。

將它連接到你的應用程式，它會監控所有應用的活動。客戶來信、GitHub 議題被指派、Sentry 發出警報、會議結束。CORE 會看到，檢查你的記憶和已設定的技能，然後自行處理或提交給你判斷。你不需要觸發它，CORE 會自動發現。

在 Claude Code、Codex 或 Cursor 安裝 CORE 插件，你的代理對話也會被監控。討論過的上下文、做出的決策、撰寫的程式碼，全部都會餵入記憶知識圖譜。CORE 接下來接手的任務，已經知道你上次會話發生了什麼。

當它執行時，就是直接執行。它可以回覆電子郵件、更新 Linear 任務、建立 GitHub PR、發送 Slack 訊息、執行終端指令、操作瀏覽器，並從任何介面啟動 Claude Code 或 Codex 工作階段。你在機場發 WhatsApp 訊息，CORE 就能在你登機前啟動程式碼工作階段並開啟 PR。
你決定它何時自行操作，何時等待你的指令。自動處理 Sentry 警報，但合併前一定先徵求你的同意。編碼會議開始前需先批准計劃。任何電子郵件發出前必須確認。自主程度由你決定，可依任務、應用程式、動作分別設定。

## 四種互動方式

聊天機器人只有一個介面。作業系統則有多種。

**語音。** 在 Mac 上按 Ctrl+Option，說出你要執行的事情。CORE 在背景自動運作，不打斷你的工作流程。

**記事板。** 打開你的每日頁面，寫下 `[ ] 修正來自 issue #47 的驗證錯誤`。CORE 在 3 分鐘內接收，從你的儲存庫與記憶體讀取上下文，草擬計劃。

**訊息。** WhatsApp、Slack、Telegram。從機場、手機、床上發送任務。CORE 會完整掌握你的上下文，不論訊息來源。

**聊天。** 打開儀表板直接對話，就像任何助理一樣。當你想在委派前進行反覆交流時。

一套 AI，四個介面，背後都是同一記憶體與上下文。

## 客製化你的 AI

給它命名。選擇它的語調。設定它在各處遵循的規則。

從五種內建個性中挑選——TARS 的冷靜效率、Alfred 的忠誠正經、Hudson 的溫暖實用——或自己編寫。個性貫穿它執行的每個任務、草擬的每個計劃、代表你發出的每則訊息。

選擇語音互動的聲音。從此，它在所有介面上都以你設定的 AI 音色和行為運作。

## CORE 實際運作

### 說出需求，回來看到 PR。

*按 Ctrl+Option，說：*「修正來自 issue #312 結帳流程的競爭狀況。」

CORE 讀取該問題，拉取相關提交紀錄與 Slack 討論，草擬計劃，並啟動 Claude Code 會話。你回來時看到差異檔。你根本不在桌前。

### 夜裡書寫，早晨審查。


*便條本:* `[ ] 今晚11點開始處理待辦清單`

CORE 從 Linear、GitHub 及記憶體中提取資料，進行優先排序，並在你睡覺時自動處理。流暢運作的流程等你審查。遇到卡住的工作，會只帶回一個明確的問題，而不是讓分頁停滯。

### 隨時隨地發送指令。

*在機場用 WhatsApp:*「部署認證重構。」

CORE 已經知道分支、上下文和你的偏好。它在 Railway 上運行。在你登機前就已啟動工作階段。

### 在警報成為事故前主動調查。

*Sentry 在凌晨2點發出警報。* CORE 會調查，從記憶體中提取相關追蹤和過往事故，提出修復方案，並在 Slack 上通知你：「問題 #847，已提出修復方案，等待你審核。」你可用手機直接核准。

### 獲得真正懂你一週的簡報。

*每日上午8點的重複任務。* CORE 從 email、GitHub、Linear、Slack 提取訊息，篩選真正需要關注的事項，自動將後續跟進變成任務，忽略不重要的內容。

## CORE 的組成

| | |
|---|---|
| **記憶體** | 橫跨所有工具與對話的時序知識圖譜。包含偏好、決策、目標和指令，每個任務都帶有完整上下文。 |
| **任務** | 一次性或重複性的工作單元，包含你的規格、CORE 的計畫、即時狀態，以及專屬聊天室。每個任務都能衍生程式碼、瀏覽器或終端機工作階段。 |
| **連接器** | 50+ 種應用透過一個 MCP 端點串接，還有 webhook 觸發主動自動化。支援 GitHub、Linear、Jira、Slack、Gmail、Calendar、Sentry、Notion、Todoist 等等。 |
| **技能** | 可重複利用的指令，依上下文自動觸發。例如：「規劃修復前一律拉取相關 Linear 問題」、「PR 前先跑測試」、「任務結束自動發 Slack 摘要」。內建100+，也可自訂。 |
| **閘道器** | 可在你的機器、Docker 或 Railway 上執行 Claude Code、Codex、瀏覽器代理及終端機指令，讓 CORE 在你筆電關機時仍可運作。 |
| **模型相容** | 可自選供應商：Anthropic、OpenAI 或開源模型。可自架全套服務，實現完全隔離。 |

## 快速開始

開源且可自行架設。您的資料保留在您的基礎設施內。

**請選擇您的路徑：**

| 我想要... | 操作方式 |
|---|---|
| 在本機嘗試 | 執行下方一鍵安裝（需 Docker） |
| 部署至伺服器或 VPS | 一鍵 Railway 部署 |
| 使用 Mac 應用程式 | [加入候補名單](https://www.getcore.me/) |

**安裝並啟動 CORE：**

```bash
npm install -g @redplanethq/corebrain && corebrain setup
```
設定精靈會要求輸入安裝目錄、AI 服務提供者、API 金鑰和聊天模型。它會產生密鑰、啟動堆疊，並開啟 `http://localhost:3033`。

大多數本地安裝只需在 Docker 執行後幾分鐘即可完成。

**或部署到 Railway：**

[![Deploy on Railway](https://railway.app/button.svg)](https://railway.com/deploy/core)

**連接閘道器**，讓 CORE 可以運行程式代理、操作您的瀏覽器並存取本機資料夾：


```bash
corebrain login
corebrain gateway setup
```

**需求:** Docker 20.10+、Docker Compose 2.20+、4 核心 CPU / 8GB 記憶體

[完整自架指南](https://docs.getcore.me/self-hosting/setup)

**您的第一個任務（設定後 2 分鐘內）：**

1. 開啟 **Scratchpad**（您每日的頁面，網址為 `http://localhost:3033`）
2. 輸入 `[ ] Summarize my open GitHub issues` 或任何您平常會自己執行的任務
3. CORE 會在 3 分鐘內自動接收，從連接的應用取得情境，並草擬計劃
4. 核准該計劃後，CORE 會執行並回傳結果

[連接您的第一個應用](https://docs.getcore.me/connectors)
