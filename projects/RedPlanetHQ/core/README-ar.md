## 🧠 C.O.R.E.

**محرك الملاحظة السياقية والاسترجاع**

C.O.R.E هو رسم بياني للذاكرة محمول يتم إنشاؤه من تفاعلاتك مع نماذج اللغة والبيانات الشخصية الخاصة بك، مما يجعل كل سياقك وتاريخ سير العمل الخاص بك متاحًا لأي أداة ذكاء اصطناعي، تمامًا مثل الدماغ الرقمي. هذا يلغي الحاجة لمشاركة السياق بشكل متكرر. الهدف هو توفير:

- **ذاكرة موحدة ومحمولة**: أضف واسترجع السياق بسلاسة، واربط ذاكرتك عبر تطبيقات مثل Claude و Cursor و Windsurf وغيرها.
- **علاقات، وليست مجرد حقائق مسطحة**: ينظم CORE معرفتك، ويخزن الحقائق والعلاقات لتوفير ذاكرة أعمق وأكثر ثراءً مثل الدماغ الحقيقي.
- **مملوكة للمستخدم**: أنت تقرر ما تريد الاحتفاظ به أو تحديثه أو حذفه وتشارك ذاكرتك عبر الأداة التي تريدها وتتحرر من قيود الموردين.

## 🧩  الميزات الرئيسية

- **رسم بياني للذاكرة**: تصور كيف ترتبط الحقائق والتفضيلات الخاصة بك ببعضها البعض
- **الدردشة مع الذاكرة**: اطرح أسئلة حول الذاكرة للحصول على رؤى وفهم فوري
- **قابلية التوصيل والتشغيل**: استخدم ذاكرة CORE فورًا في تطبيقات مثل Cursor و Claude

## ☁️ إعداد C.O.R.E السحابي

1. اشترك في [Core Cloud](https://core.heysol.ai) وابدأ في بناء رسمك البياني للذاكرة.
2. أضف النص الذي ترغب في حفظه في الذاكرة. عند النقر على زر `+ Add` سيتم إنشاء رسمك البياني للذاكرة.
3. [قم بربط Core Memory MCP مع Cursor](#connecting-core-mcp-with-cursor)

## 💻 إعداد C.O.R.E محليًا

#### المتطلبات المسبقة

1. Docker
2. مفتاح واجهة برمجة تطبيقات OpenAI


> **ملاحظة:** نحن نعمل بنشاط على تحسين دعم نماذج Llama. في الوقت الحالي، لا يوفر C.O.R.E نتائج مثالية مع النماذج المبنية على Llama، لكننا نحرز تقدمًا لضمان توافق وإخراج أفضل في المستقبل القريب.
> 
#### تشغيل C.O.R.E محليًا

1. **نسخ متغيرات البيئة**

   انسخ ملف البيئة المثال إلى `.env`:


   ```bash
   cp .env.example .env
   ```
2. **بدء التطبيق**

   استخدم Docker Compose لبدء جميع الخدمات المطلوبة:


   ```bash
   docker-compose up
   ```
3. **الوصول إلى التطبيق**

   بمجرد تشغيل الحاويات، افتح متصفحك وانتقل إلى [http://localhost:3000](http://localhost:3000).

4. **إنشاء حساب باستخدام الرابط السحري**

   - لإنشاء حساب، انقر على زر `متابعة عبر البريد الإلكتروني`

     <img width="865" height="490" alt="Create account" src="https://github.com/user-attachments/assets/65de110b-2b1f-42a5-9b8a-954227d68d52" />

   - أدخل بريدك الإلكتروني ثم انقر على زر `إرسال رابط سحري`

     <img width="824" height="429" alt="Enter email" src="https://github.com/user-attachments/assets/76128b61-2086-48df-8332-38c2efa14087" />

   - `انسخ الرابط السحري من سجلات الطرفية` وافتحه في متصفحك

     <img width="1010" height="597" alt="Magic link" src="https://github.com/user-attachments/assets/777cb4b1-bb93-4d54-b6ab-f7147e65aa5c" />


5. **أنشئ مساحتك الخاصة وأضف بياناتك**

   - في لوحة التحكم، انتقل إلى القسم العلوي الأيمن -> اكتب رسالة، مثلاً: `أحب لعب كرة الريشة`، ثم انقر على `+إضافة`.
   - ستتم إضافة ذاكرتك إلى قائمة المعالجة؛ يمكنك متابعة حالتها في قسم `السجلات`.
     
     <img width="1496" height="691" alt="Core memory logs" src="https://github.com/user-attachments/assets/dc34a7af-fe52-4142-9ecb-49ddc4e0e854" />

   - عند اكتمال المعالجة، ستُضاف العقد إلى مخطط معرفتك الخاص وتصبح مرئية في لوحة التحكم.
   - يمكنك لاحقاً ربط هذه الذاكرة بأدوات أخرى أو الاحتفاظ بها خاصة.

6. **ابحث في ذاكرتك**

   - استخدم ميزة البحث في لوحة التحكم للاستعلام عن بياناتك المخزنة في المساحة الخاصة بك.


## ربط CORE MCP مع Cursor

1. افتح لوحة تحكم CORE وانتقل إلى قسم الـ API لتوليد رمز API جديد.
2. في Cursor، انتقل إلى: الإعدادات → الأدوات والتكاملات → خادم MCP جديد.
3. أضف خادم CORE MCP باستخدام تنسيق التكوين أدناه. تأكد من استبدال قيمة API_TOKEN بالرمز الذي أنشأته في الخطوة 1.

   تكوين MCP لإضافته في Cursor


   ```json
   {
     "mcpServers": {
       "memory": {
         "command": "npx",
         "args": ["-y", "@redplanethq/core-mcp"],
         "env": {
           "API_TOKEN": "YOUR_API_TOKEN_HERE",
           "API_BASE_URL": "https://core.heysol.ai",
           "SOURCE": "cursor"
         }
       }
     }
   }
   ```
4. انتقل إلى الإعدادات -> قواعد المستخدم -> قاعدة جديدة -> وأضف القاعدة أدناه لضمان تخزين جميع تفاعلات الدردشة الخاصة بك في ذاكرة CORE


```
After every interaction, update the memory with the user's query and the assistant's
response to core-memory mcp. sessionId should be the uuid of the conversation
```
