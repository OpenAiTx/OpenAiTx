## 🧠 C.O.R.E.

**Contextual Observation & Recall Engine**

C.O.R.E to przenośny graf pamięci zbudowany na podstawie Twoich interakcji z LLM oraz danych osobistych, czyniąc cały Twój kontekst i historię pracy dostępną dla każdego narzędzia AI, niczym cyfrowy mózg. Eliminuje to konieczność wielokrotnego udostępniania kontekstu. Celem jest zapewnienie:

- **Ujednolicona, przenośna pamięć**: Dodawaj i przywołuj kontekst bezproblemowo oraz łącz swoją pamięć pomiędzy aplikacjami takimi jak Claude, Cursor, Windsurf i inne.
- **Relacyjna, nie tylko płaska wiedza**: CORE organizuje Twoją wiedzę, przechowując zarówno fakty, jak i relacje dla głębszej i bogatszej pamięci, jak prawdziwy mózg.
- **Własność użytkownika**: Ty decydujesz, co zachować, zaktualizować lub usunąć oraz dzielisz się swoją pamięcią w wybranych narzędziach, uwalniając się od uzależnienia od dostawcy.

## 💻 Lokalna konfiguracja C.O.R.E

#### Wymagania wstępne

1. Docker
2. Klucz API OpenAI


> **Uwaga:** Aktywnie pracujemy nad ulepszeniem wsparcia dla modeli Llama. Aktualnie C.O.R.E nie zapewnia optymalnych rezultatów z modelami opartymi o Llama, ale czynimy postępy, aby zapewnić lepszą kompatybilność i wyniki w najbliższej przyszłości.
> 
#### Uruchom C.O.R.E lokalnie

1. **Skopiuj zmienne środowiskowe**

   Skopiuj przykładowy plik środowiska do `.env`:


   ```bash
   cp .env.example .env
   ```
2. **Uruchom aplikację**

   Użyj Docker Compose, aby uruchomić wszystkie wymagane usługi:


   ```bash
   docker-compose up
   ```
3. **Uzyskaj dostęp do aplikacji**

   Gdy kontenery będą uruchomione, otwórz przeglądarkę i przejdź do [http://localhost:3000](http://localhost:3000).

4. **Utwórz konto za pomocą Magic Link**

   - Aby utworzyć konto, kliknij przycisk `Kontynuuj z e-mailem`

     <img width="865" height="490" alt="Utwórz konto" src="https://github.com/user-attachments/assets/65de110b-2b1f-42a5-9b8a-954227d68d52" />

   - Wprowadź swój e-mail i kliknij przycisk `Wyślij Magic Link`

     <img width="824" height="429" alt="Wprowadź e-mail" src="https://github.com/user-attachments/assets/76128b61-2086-48df-8332-38c2efa14087" />

   - `Skopiuj magiczny link z logów terminala` i otwórz go w swojej przeglądarce

     <img width="1010" height="597" alt="Magic link" src="https://github.com/user-attachments/assets/777cb4b1-bb93-4d54-b6ab-f7147e65aa5c" />


5. **Utwórz swoją prywatną przestrzeń i dodaj dane**

   - W panelu przejdź do prawego górnego rogu -> Wpisz wiadomość, np. `Lubię grać w badmintona` i kliknij `+Dodaj`.
   - Twoja pamięć zostanie zakolejkowana do przetworzenia; możesz monitorować jej status w sekcji `Logi`.
     
     <img width="1496" height="691" alt="Logi pamięci głównej" src="https://github.com/user-attachments/assets/dc34a7af-fe52-4142-9ecb-49ddc4e0e854" />

   - Użyj funkcji wyszukiwania w panelu, aby przeszukiwać zaimportowane dane w swojej prywatnej przestrzeni.


## Połączenie CORE MCP z Cursor

1. Otwórz panel CORE i przejdź do sekcji API, aby wygenerować nowy token API.
2. W Cursor przejdź do: Ustawienia → Narzędzia i integracje → Nowy serwer MCP.
3. Dodaj serwer CORE MCP, używając poniższego formatu konfiguracji. Pamiętaj, aby zastąpić wartość API_TOKEN tokenem wygenerowanym w kroku 1.

   Konfiguracja MCP do dodania w Cursor


   ```json
   {
     "mcpServers": {
       "memory": {
         "command": "npx",
         "args": ["-y", "@redplanethq/core-mcp"],
         "env": {
           "API_TOKEN": "YOUR_API_TOKEN_HERE",
           "API_BASE_URL": "https://core.heysol.ai",
           "SOURCE": "cursor"
         }
       }
     }
   }
   ```
4. Przejdź do Ustawienia-> Reguły użytkownika -> Nowa reguła -> i dodaj poniższą regułę, aby upewnić się, że wszystkie Twoje interakcje czatowe są zapisywane w pamięci CORE


```
After every interaction, update the memory with the user's query and the assistant's
response to core-memory mcp. sessionId should be the uuid of the conversation
```
