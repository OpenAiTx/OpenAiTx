## 时刻监控，随时准备

大多数 AI 工具都在等待被请求。CORE 主动监控。

将它连接到你的应用程序，它会监控所有应用的活动。客户发来邮件，GitHub 问题被分配，Sentry 警报触发，会议结束。CORE 都能看到，检查你的记忆和已设定技能，并根据情况处理或呈现给你判断。你无需触发它。它自己主动察觉。

在 Claude Code、Codex 或 Cursor 安装 CORE 插件后，你的智能体对话也会被监控。讨论的上下文、做出的决策、编写的代码，全部都被录入记忆知识图谱。CORE 执行下一个任务时，已经知晓你上一次会话发生了什么。

当它行动时，就是直接行动。它可以回复邮件、更新 Linear 问题、提交 GitHub PR、发送 Slack 消息、运行终端命令、驱动浏览器，以及从任意界面启动 Claude Code 或 Codex 会话。从机场发送 WhatsApp 消息，CORE 就能在你登机前开启编码会话并打开 PR。


你可以决定哪些操作由它自主完成，哪些操作则等待你的指令。自动处理 Sentry 报警，但每次合并前都要征求你的同意。在编码会话开始前先审批方案。任何邮件发出前都需确认。每个任务、每个应用、每个动作的自主性都由你设定。

## 四种交互方式

聊天机器人只有一个界面。操作系统有很多。

**语音。** 在 Mac 上按下 Ctrl+Option 并说出要做的事。CORE 在后台运行，不会打断你的工作流。

**便签。** 打开你的每日页面，写下 `[ ] 修复第 #47 号问题中的认证 bug`。CORE 会在 3 分钟内获取任务，从你的代码库和记忆中加载上下文，并起草方案。

**消息。** WhatsApp、Slack、Telegram。你可以在机场、手机上、床上发任务。无论消息来自哪里，CORE 都能获取你的完整上下文。

**聊天。** 打开仪表盘，直接对话，就像任何助手一样。适用于需要来回沟通再委托任务的场景。

一套 AI，四种触点，背后是统一的记忆和上下文。

## 个性化配置

给它起个名字。选择它的说话风格。设置它始终遵循的规则。

可选五种内置人格——TARS 的冷静高效，Alfred 的忠诚正式，Hudson 的温暖实用——或自定义属于你的人格。这个人格会贯穿它执行的每个任务、起草的每个方案、为你发送的每条信息。

为语音交互挑选声音。从此，它在每一个界面都以你设定的方式说话和行动。

## CORE 实际应用

### 说出任务，回来就是 PR。

*按下 Ctrl+Option，说：* “修复第 #312 号问题中结账流程的竞态条件。”

CORE 加载问题，提取相关提交和 Slack 讨论，起草方案，并运行 Claude Code 会话。你回来时已是 diff。你甚至没在工位上。

### 晚上写下，早上审查。

*便签:* `[ ] 从晚上11点开始处理今晚的积压任务`

CORE从Linear、GitHub和记忆中提取任务，优先排序，并在你睡觉时处理。顺利的运行正等待你的审核。卡住的会话只带回来一个精确问题，而不是一个停滞的标签页。

### 随时随地发送消息给它。

*机场用WhatsApp发消息:* "发布认证重构。"

CORE已经知道分支、上下文和你的偏好。它正在Railway中运行。它会在你登机前启动会话。

### 在警报变成事故前进行调查。

*Sentry凌晨2点触发。* CORE调查，提取相关的追踪和历史事件，提出修复方案，并在Slack上通知你："Issue #847，已提出修复方案，等待你的审核。"你在手机上批准。

### 获得已了解你本周情况的简报。

*每日上午8点的重复任务。* CORE从邮件、GitHub、Linear和Slack中提取，突出真正需要关注的内容，跳过无需处理的，并自动将跟进转为任务。

## CORE内部结构

| | |
|---|---|
| **记忆** | 跨所有工具和对话的时序知识图谱。偏好、决策、目标和指令，让每个任务都加载好上下文。 |
| **任务** | 一次性或重复的工作单元，包含你的规格、CORE的计划、实时状态和专属聊天线程。每个任务可生成编码、浏览器或终端会话。 |
| **连接器** | 50+应用通过一个MCP端点连接，并有Webhook触发器进行主动自动化。支持GitHub、Linear、Jira、Slack、Gmail、Calendar、Sentry、Notion、Todoist等。 |
| **技能** | 可复用指令，会根据上下文自动触发。例如："规划修复前总是提取相关的Linear问题"，"打开PR前运行测试"，或"任务完成时在Slack发布摘要"。内置100+，也可自定义。 |
| **网关** | 可在你的机器或Docker、Railway上运行Claude Code、Codex、浏览器代理和终端命令，让CORE在你笔记本关闭时持续工作。 |
| **模型无关性** | 可自选提供商：Anthropic、OpenAI或开源权重模型。可自托管全栈，实现完全隔离。 |

## 快速入门

开源且可自托管。您的数据保留在您的基础设施中。

**请选择您的路径：**

| 我想要... | 操作方法 |
|---|---|
| 在本机试用 | 运行下方一键安装命令（需要 Docker） |
| 部署在服务器或 VPS 上 | 一键部署到 Railway |
| 使用 Mac 应用程序 | [加入候补名单](https://www.getcore.me/) |

**安装并启动 CORE：**

```bash
npm install -g @redplanethq/corebrain && corebrain setup
```

设置向导会要求输入安装目录、AI 提供商、API 密钥和聊天模型。它会生成密钥，启动服务，并打开 `http://localhost:3033`。

大多数本地安装在 Docker 运行后只需几分钟。

**或者在 Railway 上部署：**

[![在 Railway 上部署](https://railway.app/button.svg)](https://railway.com/deploy/core)

**连接网关**，以便 CORE 可以运行编码代理、驱动你的浏览器并访问本地文件夹：

```bash
corebrain login
corebrain gateway setup
```
**要求：** Docker 20.10+，Docker Compose 2.20+，4 vCPU / 8GB 内存

[完整自托管指南](https://docs.getcore.me/self-hosting/setup)

**你的第一个任务（设置后2分钟内完成）：**

1. 打开 **草稿本**（你的每日页面，地址为 `http://localhost:3033`）
2. 输入 `[ ] 总结我未完成的 GitHub 问题` 或者你平时会自己完成的任何任务
3. CORE 会在3分钟内自动获取任务，收集已连接应用的上下文，并草拟执行方案
4. 审核方案后，CORE 会自动执行并返回结果

[连接你的第一个应用](https://docs.getcore.me/connectors)
