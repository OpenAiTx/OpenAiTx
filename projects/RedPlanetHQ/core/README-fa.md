# سیستم عامل هوش مصنوعی شخصی شما

یک چت‌بات که فقط بازش کنید نیست. یک هوش مصنوعی که همیشه فعال است، همیشه در حال مشاهده است.
به آن نام بدهید. آن را شکل دهید. به همه چیزهایی که استفاده می‌کنید متصلش کنید. از هر روشی که کار می‌کنید به آن دسترسی داشته باشید.
متن‌باز، خود-میزبان، همیشه مال شما.

مشاهده کنید که CORE چگونه یک وظیفه متنی ساده را می‌گیرد، زمینه را از گیت‌هاب و حافظه جمع‌آوری می‌کند، کار را برنامه‌ریزی می‌کند، یک نشست Claude Code اجرا می‌کند و یک PR باز می‌کند:

## همیشه ناظر. همیشه آماده.

بیشتر ابزارهای هوش مصنوعی منتظر درخواست شما هستند. CORE مشاهده می‌کند.

آن را به اپلیکیشن‌های خود متصل کنید تا فعالیت‌ها را در تمام آن‌ها پایش کند. یک ایمیل از مشتری می‌رسد، یک issue گیت‌هاب اختصاص داده می‌شود، یک هشدار Sentry فعال می‌شود، یک جلسه پایان می‌یابد. CORE آن را می‌بیند، حافظه و مهارت‌های تنظیم‌شده شما را بررسی می‌کند و یا آن را مدیریت می‌کند یا برای تصمیم شما نمایش می‌دهد. شما آن را فعال نمی‌کنید. خودش متوجه می‌شود.

افزونه CORE را در Claude Code، Codex یا Cursor نصب کنید تا مکالمات عامل شما هم پایش شود. زمینه‌های مورد بحث، تصمیمات گرفته‌شده، کد نوشته‌شده، همه وارد گراف دانش حافظه می‌شوند. وظیفه بعدی که CORE برمی‌دارد، قبلاً می‌داند در نشست قبلی چه رخ داده است.

هنگام عمل، مستقیماً اقدام می‌کند. می‌تواند به ایمیل‌ها پاسخ دهد، مسائل Linear را به‌روزرسانی کند، PR گیت‌هاب ثبت کند، پیام Slack بفرستد، دستورات ترمینال اجرا کند، مرورگر را کنترل کند، و یک نشست Claude Code یا Codex را از هر واسطی اجرا کند. از فرودگاه یک پیام واتساپ بفرستید و CORE می‌تواند پیش از سوار شدن شما یک نشست کدنویسی اجرا و یک PR باز کند.
شما تصمیم می‌گیرید که کجا به طور خودکار عمل کند و کجا منتظر دستور شما بماند. هشدارهای Sentry را به صورت خودکار مدیریت کند اما همیشه قبل از ادغام سؤال کند. یک طرح را قبل از شروع جلسه کدنویسی تأیید کنید. قبل از ارسال هر ایمیل تأیید بخواهید. سطح خودمختاری را می‌توانید برای هر وظیفه، هر اپلیکیشن، هر اقدام تنظیم کنید.

## چهار راه برای ارتباط با آن

یک چت‌بات فقط یک رابط دارد. یک سیستم عامل چندین رابط.

**صدا.** کلیدهای Ctrl+Option را در مک فشار دهید و بگویید چه کاری باید انجام شود. CORE آن را در پس‌زمینه اجرا می‌کند بدون اینکه روند کارتان را مختل کند.

**دفترچه یادداشت.** صفحه روزانه خود را باز کنید و بنویسید `[ ] رفع مشکل auth از issue #47`. CORE ظرف ۳ دقیقه آن را دریافت می‌کند، زمینه را از مخزن و حافظه شما بارگذاری می‌کند و یک طرح اولیه آماده می‌سازد.

**پیام‌رسانی.** واتساپ، اسلک، تلگرام. یک وظیفه را از فرودگاه، از گوشی، از تخت ارسال کنید. CORE همیشه زمینه کامل شما را دارد، فارغ از اینکه پیام از کجا می‌آید.

**چت.** داشبورد را باز کنید و مستقیم صحبت کنید، مثل هر دستیار دیگری. زمانی که قبل از واگذاری، نیاز به گفتگو دارید.

یک هوش مصنوعی، چهار سطح، حافظه و زمینه یکسان پشت همه آن‌ها.

---

## شخصی‌سازی

برای آن یک نام انتخاب کنید. نحوه صحبت کردنش را تعیین کنید. قوانین مورد نظر را در همه جا تنظیم کنید.

از پنج شخصیت داخلی انتخاب کنید — بهره‌وری خشک TARS، رسمی وفادار Alfred، عملی گرم Hudson — یا شخصیت خودتان را بنویسید. شخصیت در هر وظیفه‌ای که اجرا می‌کند، هر طرحی که تهیه می‌کند، هر پیامی که به نمایندگی از شما ارسال می‌کند، جاری است.

صدای آن را برای تعاملات گفتاری انتخاب کنید. از آن به بعد، در هر رابطی مانند هوش مصنوعی‌ای که پیکربندی کردید، رفتار و صدا خواهد داشت.

## CORE در عمل

### بگویید، به یک PR برگردید.

*کلیدهای Ctrl+Option را فشار دهید، بگویید:* «شرط مسابقه در روند پرداخت از issue #312 را رفع کن.»

CORE مسئله را بارگذاری می‌کند، کامیت‌های مرتبط و گفتگوهای اسلک را جمع‌آوری می‌کند، طرحی تهیه می‌کند و یک جلسه Claude Code اجرا می‌کند. شما به یک diff برمی‌گردید. هیچ‌وقت پشت میزتان نبودید.

### شب بنویسید، صبح مرور کنید.


*یادداشت:* `[ ] پردازش عقب‌ماندگی امشب را از ساعت ۱۱ شب شروع کن`

CORE از Linear، GitHub و حافظه داده می‌کشد، اولویت‌بندی می‌کند و زمانی که شما خواب هستید، روی آن کار می‌کند. اجراهای روان آماده بازبینی شما هستند. نشست‌های گیرکرده فقط با یک سؤال دقیق برمی‌گردند، نه یک برگه متوقف‌شده.

### از هر جا به آن پیام بدهید.

*واتساپ از فرودگاه:* "بازسازی احراز هویت را منتشر کن."

CORE از قبل شاخه، زمینه و ترجیحات شما را می‌داند. روی Railway اجرا می‌شود. جلسه را قبل از سوار شدن شما راه‌اندازی می‌کند.

### هشدارها را قبل از تبدیل به حادثه بررسی کنید.

*Sentry ساعت ۲ صبح فعال می‌شود.* CORE بررسی می‌کند، ردیابی‌های مرتبط و حوادث قبلی را از حافظه استخراج می‌کند، یک راه‌حل پیشنهاد می‌دهد و به شما در Slack پیام می‌دهد: "Issue #847، راه‌حل پیشنهاد شد، منتظر بازبینی شماست." شما از روی گوشی تأیید می‌کنید.

### خلاصه‌ای بگیرید که از هفته شما باخبر است.

*کار تکراری، هر روز صبح ساعت ۸.* CORE از ایمیل، GitHub، Linear و Slack داده می‌کشد، مواردی که واقعاً نیاز به توجه دارند را سطح‌بندی می‌کند، موارد غیرضروری را رد می‌کند و پیگیری‌ها را به طور خودکار به کار تبدیل می‌کند.

## چه چیزی داخل CORE است

| | |
|---|---|
| **حافظه** | نمودار دانش زمانی در تمام ابزارها و گفتگوها. ترجیحات، تصمیمات، اهداف و دستورات، تا هر کار با زمینه بارگذاری شده شروع شود. |
| **کارها** | واحدهای کاری یک‌باره یا تکراری با مشخصات شما، طرح CORE، وضعیت زنده و یک رشته گفتگوی اختصاصی. هر کار می‌تواند نشست کدنویسی، مرورگر یا ترمینال ایجاد کند. |
| **اتصالات** | بیش از ۵۰ اپلیکیشن از طریق یک نقطه انتهایی MCP، به‌علاوه تریگرهای webhook برای خودکارسازی فعال. GitHub، Linear، Jira، Slack، Gmail، Calendar، Sentry، Notion، Todoist و غیره. |
| **مهارت‌ها** | دستورالعمل‌های قابل استفاده مجدد که به طور خودکار بر اساس زمینه اجرا می‌شوند. برای مثال: "همیشه پیش از برنامه‌ریزی راه‌حل، مسائل مرتبط Linear را بکش"، "پیش از باز کردن PR تست اجرا کن"، یا "وقتی یک کار کامل شد خلاصه را در Slack ارسال کن." بیش از ۱۰۰ داخلی، یا خودتان بنویسید. |
| **درگاه** | اجرای Claude Code، Codex، ایجنت‌های مرورگر و دستورات ترمینال روی دستگاه شما یا در Docker و Railway، بنابراین CORE حتی وقتی لپ‌تاپ شما بسته است کار می‌کند. |
| **مدل مستقل** | ارائه‌دهنده خود را بیاورید: Anthropic، OpenAI یا مدل‌های open-weight. کل پشته را به صورت مستقل میزبانی کنید. |

## شروع سریع

متن‌باز و خودمیزبان. داده‌های شما در زیرساخت خودتان باقی می‌ماند.

**مسیر خود را انتخاب کنید:**

| می‌خواهم... | چگونه |
|---|---|
| امتحانش کنم روی دستگاه خودم | نصب یک‌مرحله‌ای زیر را اجرا کنید (نیاز به Docker دارد) |
| استقرار روی سرور یا VPS | استقرار تک‌کلیک با Railway |
| استفاده از اپ مک | [به لیست انتظار بپیوندید](https://www.getcore.me/) |

**نصب و شروع CORE:**

```bash
npm install -g @redplanethq/corebrain && corebrain setup
```

ویزارد راه‌اندازی از شما می‌خواهد که مسیر نصب، ارائه‌دهنده هوش مصنوعی، کلید API و مدل گفتگو را وارد کنید. این ویزارد رمزها را تولید کرده، استک را راه‌اندازی می‌کند و آدرس `http://localhost:3033` را باز می‌کند.

بیشتر نصب‌های محلی پس از اجرای داکر چند دقیقه زمان می‌برند.

**یا روی Railway مستقر کنید:**

[![استقرار روی Railway](https://railway.app/button.svg)](https://railway.com/deploy/core)

**یک دروازه متصل کنید** تا CORE بتواند عوامل کدنویسی را اجرا کند، مرورگر شما را کنترل کند و به پوشه‌های محلی دسترسی پیدا کند:

```bash
corebrain login
corebrain gateway setup
```
**نیازمندی‌ها:** داکر ۲۰.۱۰+، داکر کامپوز ۲.۲۰+، ۴ vCPU / ۸ گیگابایت رم

[راهنمای کامل میزبانی شخصی](https://docs.getcore.me/self-hosting/setup)

**اولین کار شما (۲ دقیقه پس از راه‌اندازی):**

۱. **دفترچه یادداشت** را باز کنید (صفحه روزانه شما در `http://localhost:3033`)
۲. بنویسید `[ ] خلاصه کردن مسائل باز گیت‌هاب من` یا هر کاری که معمولاً خودتان انجام می‌دهید
۳. CORE آن را ظرف ۳ دقیقه دریافت می‌کند، زمینه را از برنامه‌های متصل جمع‌آوری کرده و یک برنامه پیش‌نویس می‌کند
۴. برنامه را تایید کنید و CORE آن را اجرا کرده و نتیجه را بازمی‌گرداند

[اولین برنامه خود را متصل کنید](https://docs.getcore.me/connectors)

- [**تغییرات**](https://docs.getcore.me/opensource/changelog) - آنچه منتشر شده است
