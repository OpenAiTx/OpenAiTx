## 🧠 C.O.R.E.

**موتور مشاهده و یادآوری مبتنی بر زمینه**

C.O.R.E یک گراف حافظه قابل حمل است که از تعاملات شما با مدل‌های زبانی بزرگ و داده‌های شخصی‌تان ساخته می‌شود و تمام تاریخچه زمینه و جریان کاری شما را برای هر ابزار هوش مصنوعی قابل دسترس می‌کند، درست مانند یک مغز دیجیتال. این کار نیاز به اشتراک‌گذاری مکرر زمینه را از بین می‌برد. هدف ارائه موارد زیر است:

- **حافظه یکپارچه و قابل حمل**: به راحتی زمینه را اضافه و بازیابی کنید و حافظه خود را میان برنامه‌هایی مانند Claude، Cursor، Windsurf و غیره متصل نمایید.
- **روابطی، نه فقط حقایق ساده**: CORE دانش شما را سازماندهی کرده و هم حقایق و هم روابط را برای حافظه‌ای عمیق‌تر و غنی‌تر مانند یک مغز واقعی ذخیره می‌کند.
- **مالکیت کاربر**: شما تصمیم می‌گیرید چه چیزی را نگه دارید، به‌روزرسانی یا حذف کنید و حافظه خود را میان ابزارهای مورد نظر به اشتراک بگذارید و از قفل شدن در یک ارائه‌دهنده خاص آزاد باشید.

## 🧩  ویژگی‌های کلیدی

- **گراف حافظه**: مشاهده بصری ارتباط حقایق و ترجیحات شما با یکدیگر
- **گفتگو با حافظه**: پرسیدن سوالات درباره حافظه برای دریافت بینش و درک فوری
- **قابلیت اتصال سریع**: استفاده فوری از حافظه CORE در برنامه‌هایی مانند Cursor، Claude

## ☁️ راه‌اندازی C.O.R.E ابری

1. در [Core Cloud](https://core.heysol.ai) ثبت‌نام کنید و شروع به ساخت گراف حافظه خود نمایید.
2. متنی که می‌خواهید در حافظه ذخیره کنید را اضافه کنید. پس از کلیک روی دکمه `+ Add` گراف حافظه شما تولید خواهد شد.
3. [اتصال حافظه Core MCP به Cursor](#connecting-core-mcp-with-cursor)

## 💻 راه‌اندازی C.O.R.E محلی

#### پیش‌نیازها

1. Docker
2. کلید API سرویس OpenAI


> **توجه:** ما به طور فعال در حال بهبود پشتیبانی از مدل‌های Llama هستیم. در حال حاضر، C.O.R.E نتایج بهینه‌ای با مدل‌های مبتنی بر Llama ارائه نمی‌دهد، اما در تلاشیم تا به زودی سازگاری و خروجی بهتری فراهم کنیم.
> 
#### اجرای C.O.R.E به صورت محلی

1. **کپی متغیرهای محیطی**

   فایل نمونه متغیرهای محیطی را به `.env` کپی کنید:


   ```bash
   cp .env.example .env
   ```
2. **برنامه را اجرا کنید**

   برای راه‌اندازی همه سرویس‌های مورد نیاز از Docker Compose استفاده کنید:


   ```bash
   docker-compose up
   ```
3. **دسترسی به برنامه**

   پس از اجرای کانتینرها، مرورگر خود را باز کرده و به [http://localhost:3000](http://localhost:3000) بروید.

4. **ساخت حساب با Magic Link**

   - برای ساخت حساب، روی دکمه `ادامه با ایمیل` کلیک کنید

     <img width="865" height="490" alt="Create account" src="https://github.com/user-attachments/assets/65de110b-2b1f-42a5-9b8a-954227d68d52" />

   - ایمیل خود را وارد کرده و روی دکمه `ارسال Magic Link` کلیک کنید

     <img width="824" height="429" alt="Enter email" src="https://github.com/user-attachments/assets/76128b61-2086-48df-8332-38c2efa14087" />

   - `لینک جادویی را از لاگ‌های ترمینال کپی کنید` و آن را در مرورگر خود باز کنید

     <img width="1010" height="597" alt="Magic link" src="https://github.com/user-attachments/assets/777cb4b1-bb93-4d54-b6ab-f7147e65aa5c" />


5. **ایجاد فضای خصوصی و افزودن داده**

   - در داشبورد، به بخش بالا سمت راست بروید -> یک پیام وارد کنید، مثلاً `من عاشق بازی بدمینتون هستم` و روی `+افزودن` کلیک کنید.
   - حافظه شما در صف پردازش قرار می‌گیرد؛ می‌توانید وضعیت آن را در بخش `لاگ‌ها` مشاهده کنید.
     
     <img width="1496" height="691" alt="Core memory logs" src="https://github.com/user-attachments/assets/dc34a7af-fe52-4142-9ecb-49ddc4e0e854" />

   - پس از اتمام پردازش، نودها به گراف دانش خصوصی شما افزوده شده و در داشبورد قابل مشاهده خواهند بود.
   - بعداً می‌توانید این حافظه را به ابزارهای دیگر متصل کنید یا آن را خصوصی نگه دارید.

6. **جستجو در حافظه شما**

   - از قابلیت جستجوی داشبورد برای پرس‌وجو از داده‌های واردشده در فضای خصوصی خود استفاده کنید.


## اتصال CORE MCP به Cursor

1. داشبورد CORE را باز کرده و به بخش API بروید تا یک توکن API جدید ایجاد کنید.
2. در Cursor به مسیر: Settings → Tools & Integrations → New MCP Server بروید.
3. سرور CORE MCP را با استفاده از فرمت پیکربندی زیر اضافه کنید. حتماً مقدار API_TOKEN را با توکنی که در مرحله 1 ساختید جایگزین کنید.

   پیکربندی MCP برای افزودن به Cursor


   ```json
   {
     "mcpServers": {
       "memory": {
         "command": "npx",
         "args": ["-y", "@redplanethq/core-mcp"],
         "env": {
           "API_TOKEN": "YOUR_API_TOKEN_HERE",
           "API_BASE_URL": "https://core.heysol.ai",
           "SOURCE": "cursor"
         }
       }
     }
   }
   ```
4. به تنظیمات بروید -> قوانین کاربر -> قانون جدید -> و قانون زیر را اضافه کنید تا مطمئن شوید تمام تعاملات چت شما در حافظه CORE ذخیره می‌شوند.


```
After every interaction, update the memory with the user's query and the assistant's
response to core-memory mcp. sessionId should be the uuid of the conversation
```
