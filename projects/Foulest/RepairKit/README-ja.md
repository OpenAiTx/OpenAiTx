## 機能

### **自動修復**

包括的なシステムクリーンアップと修復を自動的に実行します。内容は以下の通りです：

- システムの復元ポイントの作成
- 悪意のあるシステムポリシーの削除
- レジストリとシステムの調整の実行
- 脆弱なWindows機能の削除
- 最適なWindows Defenderの設定適用
- 見つけにくいジャンクファイルのクリーンアップ
- プレインストールされた不要ソフトの削除
- ディスクの問題と破損したWindowsファイルの修復
- 古くなったインストール済みプログラムの更新
修復の各機能は完全にカスタマイズ可能で、パネルのチェックボックスまたは**config**フォルダー内の設定ファイルを使って必要に応じて特定の修復を有効または無効にできます。**これらのファイルは、内容を理解している場合のみ変更してください。**

> **注意：** 以前利用可能だった**マルウェア除去**オプションは削除されました。代わりに**Windows Defenderの調整を実行**が最適なWindows Defenderの設定を適用します。手動スキャンはUseful ProgramsパネルのEmsisoftまたはSophosスキャンオプションで利用可能です。

- **[Everything](https://voidtools.com)**：超高速のファイル検索エンジンです。
- **[CrystalDiskInfo](https://sourceforge.net/projects/crystaldiskinfo)**：ディスクドライブの健康状態とステータスを監視します。
- **[CrystalDiskMark](https://crystalmark.info/en/software/crystaldiskmark)**：ディスクドライブの読み書き速度をベンチマークします。
- **[Autoruns](https://learn.microsoft.com/en-us/sysinternals/downloads/autoruns)**：Windowsのスタートアップマネージャーの代替ツールです。
- **[Process Explorer](https://learn.microsoft.com/en-us/sysinternals/downloads/process-explorer)**：Windowsタスクマネージャーの代替ツールです。
- **[Process Monitor](https://learn.microsoft.com/en-us/sysinternals/downloads/procmon)**：システム活動とプロセスを監視します。
- **[BlueScreenView](https://nirsoft.net/utils/blue_screen_view.html)**：WindowsのBSODクラッシュダンプを表示・分析します。
- **[NVCleanstall](https://techpowerup.com/download/techpowerup-nvcleanstall)**：軽量のNVIDIAグラフィックスカードドライバーアップデーターです。
- **[DisplayDriverUninstaller](https://guru3d.com/files-details/display-driver-uninstaller-download.html)**：ディスプレイドライバーとパッケージを削除します。

## ダウンロードと実行

RepairKit のダウンロードと実行方法がわからない場合は、以下の手順に従ってください：

1. 最新版の RepairKit を
   [Releases ページ](https://github.com/Foulest/RepairKit/releases/latest) からダウンロードします（`RepairKit-X.X.X.zip` ファイルをクリック）。
2. ZIP ファイルをコンピュータのフォルダーに解凍します。[7-Zip](https://7-zip.org) などのプログラムを使うと解凍が簡単です。
3. `RepairKit-X.X.X.exe` ファイルをダブルクリックしてプログラムを実行します。
4. Windows Defender SmartScreen がアプリをブロックした場合は、「詳細情報」をクリックしてから「実行」を選択します。
5. ユーザーアカウント制御のプロンプトで「はい」をクリックします。

これで RepairKit が起動し、機能を使い始めることができます。


## コンパイル

1. リポジトリをクローンします。  
2. コマンドプロンプト／ターミナルをリポジトリのディレクトリで開きます。  
3. Windowsでは `gradlew createExe` を、macOSまたはLinuxでは `./gradlew createExe` を実行します。  
4. ビルドされた `RepairKit-X.X.X.zip` ファイルは `build` フォルダにあります。
