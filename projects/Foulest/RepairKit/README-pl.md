## Funkcje

### **Automatyczne Naprawy**

Automatycznie wykonuje kompleksowe czyszczenie i naprawę systemu, w tym:

- Tworzenie Punktu Przywracania Systemu
- Usuwanie złośliwych polityk systemowych
- Wykonywanie poprawek rejestru i systemu
- Usuwanie podatnych funkcji Windows
- Stosowanie optymalnych ustawień Windows Defendera
- Czyszczenie trudno dostępnych plików śmieci
- Usuwanie preinstalowanego bloatware
- Naprawa problemów z dyskiem i uszkodzonych plików Windows
- Aktualizacja przestarzałych programów

Każda funkcja naprawy jest w pełni konfigurowalna, co pozwala włączać lub wyłączać konkretne naprawy według potrzeb,
używając pól wyboru na panelu lub plików konfiguracyjnych znajdujących się w folderze **config**. **Modyfikuj te pliki
tylko jeśli wiesz, co robisz.**

> **Uwaga:** Poprzednia opcja **Usuń złośliwe oprogramowanie** została usunięta. **Uruchom optymalizację Windows Defendera** teraz
> stosuje optymalne ustawienia Windows Defendera. Ręczne opcje skanowania są dostępne przez Emsisoft lub Sophos
> w panelu Przydatnych Programów.

![Automatyczne Naprawy](https://i.imgur.com/mnkUQV1.png)

Uzyskaj dostęp do niezbędnych narzędzi programowych do konserwacji systemu, w tym:

## Pobieranie i uruchamianie

Nie wiesz, jak pobrać i uruchomić RepairKit? Wykonaj poniższe kroki:

1. Pobierz najnowszą wersję RepairKit ze
   [strony wydań](https://github.com/Foulest/RepairKit/releases/latest) (kliknij plik `RepairKit-X.X.X.zip`).
2. Rozpakuj plik ZIP do folderu na swoim komputerze. Możesz użyć programu takiego jak [7-Zip](https://7-zip.org), aby rozpakować zawartość.
3. Kliknij dwukrotnie plik `RepairKit-X.X.X.exe`, aby uruchomić program.
4. Jeśli Windows Defender SmartScreen zablokuje aplikację, kliknij `Więcej informacji`, a następnie `Uruchom mimo to`.
5. Kliknij `Tak` w oknie Kontroli Konta Użytkownika.

RepairKit zostanie teraz uruchomiony i możesz rozpocząć korzystanie z jego funkcji.

## Kompilacja

1. Sklonuj repozytorium.
2. Otwórz wiersz poleceń/terminal w katalogu repozytorium.
3. Uruchom `gradlew createExe` w Windows lub `./gradlew createExe` w macOS albo Linux.
4. Zbudowany plik `RepairKit-X.X.X.zip` będzie w folderze `build`.
