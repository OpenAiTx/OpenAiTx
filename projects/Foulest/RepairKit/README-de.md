## Funktionen

### **Automatische Reparaturen**

Führt automatisch eine umfassende Systembereinigung und Reparatur durch, einschließlich:

- Erstellen eines Systemwiederherstellungspunkts
- Löschen schädlicher Systemrichtlinien
- Ausführen von Registry- und Systemoptimierungen
- Entfernen anfälliger Windows-Funktionen
- Anwenden optimaler Windows Defender-Einstellungen
- Bereinigen schwer auffindbarer Junk-Dateien
- Entfernen vorinstallierter Bloatware
- Reparieren von Festplattenproblemen und beschädigten Windows-Dateien
- Aktualisieren veralteter installierter Programme

Jede Funktion der Reparatur ist vollständig anpassbar, sodass Sie bestimmte Reparaturen nach Bedarf entweder über die
Kontrollkästchen im Panel oder die Konfigurationsdateien im **config**-Ordner aktivieren oder deaktivieren können. **Ändern Sie diese
Dateien nur, wenn Sie wissen, was Sie tun.**

> **Hinweis:** Die zuvor verfügbare Option **Remove Malware** wurde entfernt. **Windows Defender Tweaks ausführen** wendet nun
> stattdessen optimale Windows Defender-Einstellungen an. Manuelle Scan-Optionen sind über die Emsisoft- oder Sophos-
> Scan-Optionen im Bereich Nützliche Programme verfügbar.

## Herunterladen und Ausführen

Nicht sicher, wie Sie RepairKit herunterladen und ausführen? Folgen Sie diesen Schritten:

1. Laden Sie die neueste Version von RepairKit von der
   [Release-Seite](https://github.com/Foulest/RepairKit/releases/latest) herunter (klicken Sie auf die Datei `RepairKit-X.X.X.zip`).
2. Entpacken Sie die ZIP-Datei in einen Ordner auf Ihrem Computer. Sie können ein Programm wie [7-Zip](https://7-zip.org) zum Entpacken verwenden.
3. Doppelklicken Sie auf die Datei `RepairKit-X.X.X.exe`, um das Programm zu starten.
4. Falls der Windows Defender SmartScreen die App blockiert, klicken Sie auf `Weitere Informationen` und dann auf `Trotzdem ausführen`.
5. Klicken Sie auf `Ja` in der Benutzerkontensteuerungs-Abfrage.

RepairKit wird nun geöffnet und Sie können die Funktionen verwenden.

## Kompilieren

1. Klonen Sie das Repository.
2. Öffnen Sie eine Eingabeaufforderung/Terminal im Repository-Verzeichnis.
3. Führen Sie `gradlew createExe` unter Windows oder `./gradlew createExe` unter macOS oder Linux aus.
4. Die erstellte Datei `RepairKit-X.X.X.zip` befindet sich im Ordner `build`.
