## 💿 Установка

Достаточно использовать **любой один** из следующих методов

> [!TIP]
> Для активации функции преобразования Markdown в изображение требуется установить `nonebot-plugin-deepseek[image]`

<details open>
<summary>[Рекомендуется] Установка с помощью nb-cli</summary>
Откройте командную строку в корневой папке бота и введите следующую команду для установки

```shell
nb plugin install nonebot-plugin-deepseek
```

</details>
<details>
<summary>Установка с помощью менеджера пакетов</summary>

```shell
pip install nonebot-plugin-deepseek
# or, use poetry
poetry add nonebot-plugin-deepseek
# or, use pdm
pdm add nonebot-plugin-deepseek
```

Откройте файл конфигурации в корневом каталоге проекта NoneBot и добавьте в раздел `[plugin]`

```toml
plugins = ["nonebot_plugin_deepseek"]
```
</details>

## ⚙️ Конфигурация

Добавьте конфигурации из таблицы ниже в файл конфигурации проекта

> [!note]
> `api_key` получите на [DeepSeek Открытой платформе](https://platform.deepseek.com/)  
> `enable_models` — словарь в структуре [`CustomModel`](https://github.com/KomoriDev/nonebot-plugin-deepseek/blob/ee9f0b0f0568eedb3eb87423e6c1bf271787ab76/nonebot_plugin_deepseek/config.py#L34), если нет необходимости подключать локальные модели, изменять не нужно  
> Для подключения локальных моделей смотрите: 👉 [Wiki — Подключение локальных моделей](https://github.com/KomoriDev/nonebot-plugin-deepseek/wiki/本地模型)  

|           Параметр конфигурации           | Обязателен |                           Значение по умолчанию                          |                                                                     Описание                                                                      |
| :---------------------------------------: | :--------: | :-----------------------------------------------------------------------: | :-----------------------------------------------------------------------------------------------------------------------------------------------: |
|        deepseek__api_key                  |    Да      |                                 Нет                                      |                                                                     API Key                                                                      |
|     deepseek__enable_models               |    Нет     | [{ "name": "deepseek-v4-flash" }, { "name": "deepseek-v4-pro" }]         | Включенные модели [Описание конфигурации](https://github.com/KomoriDev/nonebot-plugin-deepseek/wiki/%E9%85%8D%E7%BD%AE#enable_models-%E9%85%8D%E7%BD%AE%E8%AF%B4%E6%98%8E) |
|         deepseek__prompt                  |    Нет     |                                 Нет                                      |                                                                  Пресет модели                                                                  |

## 🎉 Использование

> [!note]
> Проверьте ваш `COMMAND_START` и вышеуказанные параметры. Здесь по умолчанию используется `/`

### Помощь


```bash
deepseek --help
         --balance

         [...content] 
           --use-model [model]
           --with-context
           --render | -r

         model
           --list | -l
           --set-default [model]
           --render-markdown ['enable' | 'disable' | 'on' | 'off']
```

### 问答

```bash
/deepseek [内容]
```

Быстрая команда: `/ds [содержимое]` или ответ на текстовое сообщение

### Многоступенчатый диалог

```bash
/deepseek --with-context [内容]
```

Быстрая команда: `/ds --with-context [содержимое]` `/многоходовой-диалог`

### Глубокое мышление

```bash
/deepseek [内容] --use-model deepseek-v4-pro
```

快捷指令：`/深度思考 [内容]`

### 余额

> 权限：SUPERUSER

```bash
/deepseek --balance
```

快捷指令：`/ds --balance` `/余额`

### 设置

> 权限：`设置默认模型` 指令仅 SUPERUSER 可用

```bash
# 查看支持的模型列表
/deepseek model -l|--list
# 设置默认模型
/deepseek model --set-default [模型名]
# 设置 Markdown 转图片
/deepseek model --render-markdown ['enable' | 'disable' | 'on' | 'off']
```

Быстрые команды: `/модельный_список` `/установить_модель_по_умолчанию [имя_модели]`

### Пользовательские быстрые команды

> Эта функция зависит от [Alconna быстрых команд](https://nonebot.dev/docs/2.3.3/best-practice/alconna/command#command%E7%9A%84%E4%BD%BF%E7%94%A8). Пользовательские команды не содержат `COMMAND_START`, при необходимости требуется указать вручную

```bash
# 增加
/deepseek --shortcut <自定义指令> /deepseek
# 删除
/deepseek --shortcut delete <自定义指令>
# 列出
/deepseek --shortcut list
```

例子:

```bash
user: /deepseek --shortcut /chat /deepseek --use-model deepseek-v4-flash
bot: deepseek::deepseek 的快捷指令: "/chat" 添加成功
user: /chat
bot: (使用模型 deepseek-v4-flash)
```
