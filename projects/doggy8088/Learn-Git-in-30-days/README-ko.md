# 30일 만에 마스터하는 Git 버전 관리

저자: 황보흡 (Will 보哥)

![Git](https://raw.githubusercontent.com/doggy8088/Learn-Git-in-30-days/master/zh-tw/figures/README/01.png)

## 글 소개

소프트웨어 개발 분야에서 소스 코드를 버전 관리하는 것은 매우 중요한 일입니다. Subversion이나 TFS와 같은 중앙집중식 버전 관리 시스템과 달리, Git은 분산형 버전 관리 시스템으로, 버전 관리에 있어 다양한 이점과 전통적인 중앙집중식 버전 관리의 단점을 해결해 줍니다. 예를 들어, 로컬 작업 지원, 쉬운 백업, 강력하고 유연한 브랜치 및 병합 기능 등이 있습니다. 하지만 Git 버전 관리는 개념과 도구 사용 모두 전통적인 중앙집중식 버전 관리 도구와 큰 차이가 있어 진입 장벽이 높습니다. 앞으로 30일 동안 Git 버전 관리의 핵심을 완전히 이해할 수 있도록 도우며, 저의 학습 경험과 저희 회사가 Subversion에서 Git으로 전환하는 과정에서 겪은 문제와 해결 방법을 정리했습니다.
