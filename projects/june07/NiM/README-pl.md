## Łatwiejsze niż `about::inspect (chrome://inspect/#devices)`

NiM zarządza cyklem życia okna/karty Chrome DevTools, pozwalając Ci skupić się na tym, co ważne... debugowaniu kodu.  Nie musisz już kopiować/wklejać adresów DevTools ani otwierać/zamykać kolejnych kart/okien.

NiM automatycznie wykrywa URL generowany podczas uruchamiania node (lokalnie lub zdalnie) z opcją --inspect. NiM daje możliwość automatycznego otwierania i zamykania Chrome DevTools w zakładce lub oknie. Po prostu przełącz ustawienie Manual/Auto i rozpocznij sesję debugowania.  DevTools otworzy się po kliknięciu "Otwórz DevTools" lub po upływie określonego czasu.  Jeśli ustawisz auto-zamykanie, po zakończeniu sesji debugowania karta/okno DevTools zamknie się automatycznie.
 
 * Zarządzaj i monitoruj lokalne oraz zdalne sesje debugowania
 * Ręczna lub automatyczna kontrola interfejsu DevTools
 * Otwieraj DevTools w nowej karcie lub oknie
 * Spraw, by DevTools był aktywny lub nieaktywny na starcie
 * Dostosuj czas pomiędzy sondami v8 Inspector
 * Automatyczne zapisywanie ustawień
 * Debuguj procesy node uruchamiane przez [VSCode](https://imgur.com/download/PRMn9md)
